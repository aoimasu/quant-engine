# QE-456 — `GET /api/vintages/{id}` vintage-detail read endpoint  *(R1 — inspection)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#7-vintage-inspector--apivintagesid-surface) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R1)` · `Area: backend / api` · `Depends on: QE-257, QE-260`

**Why.** The ensemble composition, per-chromosome weights, and the G1 gate/deflation evidence **already exist**
in the sealed vintage (`VintageContent.chromosomes`/`weights`/`lineage`, `crates/vintage/src/lib.rs:45`) and the
train run's gate snapshot — but `GET /api/vintages` (`crates/server/src/read.rs:68`) returns only a thin list
(`VintageListItem{id,label,summary}`) and there is **no detail endpoint**. `evolve` got a rich formula-pool
inspection (`GET /api/formula-pools/{id}`, PoolReview); the `train`/vintage path never did. This is the read
half of closing that coverage gap. Additive — **no engine change**.

**Scope / requirements.**
- Add `GET /api/vintages/{id}` beside the existing list, loading via `VintageRepository::load` (hash-verified,
  `crates/vintage/src/lib.rs:256`) on a `spawn_blocking` worker (as the list already does), session-authed.
- Response body carries:
  - **Ensemble composition** — for each `chromosomes[i]`: the referenced indicator ids
    (`Genome::referenced_features` resolved through the sealed `CatalogueIdentity`, distinguishing catalogue vs
    evolved formulas) and the aligned **weight** `weights[i]`.
  - **G1 gate / deflation snapshot** — the run's gate evidence (DSR, uncensored PBO + population size, SPA p,
    IC/FDR, distinct-trial basis, cost-stress / turnover / capacity results) as recorded by `evaluate_g1` /
    the train gate output.
  - **The frozen-holdout split** — `{holdout_range, embargo, train_range}` read from `VintageContent.lineage`
    (design §4).
  - The provenance sidecars already in the content: `slippage`, `sizer`, `worst_case_loss`, `calibration`,
    `catalogue` identity.
- `404` for an unknown id; the same error-body shape the read module already uses.

**Out of scope.** The SPA screen (QE-457). Any recomputation of gate stats (read-only over what was sealed —
never re-runs the gate). Any promote/seal/select affordance (a vintage is already sealed by the train gate;
this endpoint is inspection only, per the design's leaderboard-is-not-selection boundary). Ingest/flow work.

**Acceptance criteria.**
- `GET /api/vintages/{id}` returns composition (chromosomes→indicators + weights), the G1 gate/deflation
  snapshot, and the holdout split for a sealed vintage; the artefact is hash-verified on load.
- No new `qe-server → qe-wfo`/`qe-ensemble` code edge is introduced (the server already depends on
  `qe-vintage`; this only *reads* the sealed artefact); the `firewall`/`dependency_topology` test stays green.
- An unknown id yields `404`; a corrupt/failing-verify artefact yields a `500` with a message, never a panic.
- The endpoint performs no gate recomputation and exposes no seal/promote/select action.
