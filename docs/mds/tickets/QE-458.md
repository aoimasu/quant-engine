# QE-458 — Steerable-search params + the gate-monotone whitelist guardrail  *(R1 — steering)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#6-the-steer-knob-whitelist--the-anti-overfitting-guardrail) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R1)` · `Area: run-protocol / server / wfo` · `Depends on: QE-467, QE-260, QE-437, QE-439, QE-451`

**Why.** The owner wants to *steer* the existing (already-deflated) in-run search — "search harder, over these
indicators, over more windows" — without editing config files. `TrainParams`
(`crates/run-protocol/src/lib.rs:358`) already exposes `generations`/`population`/`holdout`/`embargo`/`seed`
but has **no** indicator-subset or window/fold control. The danger: a steer knob that quietly relaxes the G1
gate re-introduces the overfitting QE-430..454 killed. This ticket adds the steer knobs **and** the compiled
**whitelist guardrail** that proves no steer knob can move a candidate from G1-reject to G1-seal.

**Scope / requirements.**
- Extend `TrainParams` + `validate_train` (`crates/server/src/runs/manager.rs:415`) with the **whitelisted,
  gate-monotone** steer knobs (design §6.1):
  - **Search budget** — `generations` / `population` (exist; surface them as steerable).
  - **Indicator subset** — which catalogue indicators and which **already-sealed** evolved-pool formulas
    (QE-451) are in play. Consuming an evolved pool cannot un-seal or re-deflate it.
  - **Window/fold config** — number & length of WFO windows/folds.
- **(a) Indicator-subset cardinality enters the distinct-trial basis.** The **available-feature-space size** —
  catalogue-indicator count **plus** the count of included evolved-pool formulas — MUST feed the distinct-trial
  basis `N` / `E[maxSharpe]`. Extend QE-439's effective-trials computation to **ingest the available-feature-space
  size**, not only gens × windows × niches: a search allowed to reference more indicators has explored a larger
  hypothesis space and must be deflated against it.
- **(b) Deflation-SCALING property (replaces a plain monotonicity proof).** As subset cardinality / search budget
  rise, the **recorded `N` and the deflation bar are non-decreasing** — steering more can only *raise* the bar.
  PLUS a **false-discovery test on noise series**: on pure-noise inputs, enlarging the indicator subset MUST NOT
  raise the seal rate (a bigger subset cannot manufacture more seals on noise). Both are merge-gate tests. A knob
  that fails either is not admitted to the whitelist.
- **(c) Archive-coverage recording + floor (preserve the QD mandate).** Record **archive coverage** —
  occupied-niche count / descriptor-space size — **pre and post steer**, so a steer that collapses the MAP-Elites
  archive to a few niches is visible; enforce a **floor on minimum occupied niches** so steering cannot flatten
  the quality-diversity archive (preserving `specs.md`'s QD mandate). Coverage collapse is surfaced, not hidden.
- **(d) Regime-coverage invariant under steering.** The window/fold knobs **cannot shrink the total OOS span
  below a floor** or exclude the **mandated stress regime** — regime coverage is invariant to steering. A
  window/fold configuration that would drop below the OOS-span floor or omit the required stress regime is
  rejected (`400`).
- **(e) Record the steer delta into content-addressed lineage.** The applied steer (indicator-subset hash,
  gens/pop, window/fold counts) is written into `VintageContent.lineage` **via QE-467's schema** (this ticket
  *populates* the steer-delta fields QE-467 defined; it does **not** bump the version).
- Enforce the **blocklist** (design §6.2) in `validate_train`: reject (`400`) any request that sets a
  **cost-stress multiplier / max-turnover cap / capacity floor / DSR-PBO cutoff / IC-FDR threshold / holdout
  size / embargo / purge** below its compiled floor. These are **not** steerable. The `DEFLATION_BASIS_VERSION`
  const (QE-454) and `G1Criteria` (`crates/gate`, wired at G1 via QE-437) stay server-side and non-editable.
- **(f) Firewall-green load path for evolved-pool-as-indicator.** Consuming an already-sealed evolved pool as an
  indicator source must introduce **no new cross-crate code edge** that violates the search⊥portfolio firewall;
  add an AC that the `firewall`/`dependency_topology` test stays green over the evolved-pool load path.
- **The proof obligation (§6.3):** on a fixed seed/dataset where the **un-steered** gate rejects, sweep every
  whitelisted knob across its allowed range and assert **no steered configuration seals a vintage the un-steered
  gate would reject** — i.e. steering is gate-monotone. This is a merge gate (and the base case of the (b)
  scaling property).

**Out of scope.** The SPA controls (QE-459). The composite flow (QE-460). Adding any new gate threshold or
changing `evaluate_g1`. Authoring evolved formulas (that stays the `evolve` campaign, QE-451/452). Defining the
lineage schema (that is QE-467 — this ticket only populates the steer-delta fields).

**Acceptance criteria.**
- `TrainParams`/`validate_train` accept the whitelisted steer knobs (budget, indicator subset, windows/folds);
  the wire shape stays `#[serde(default)]`-lenient with uniform `400` validation.
- The indicator-subset cardinality (catalogue count + included evolved-pool-formula count) feeds `N` /
  `E[maxSharpe]` through the extended QE-439 effective-trials computation.
- The **deflation-scaling** test passes: raising subset cardinality / budget is non-decreasing in recorded `N`
  and the deflation bar; and a **noise-series false-discovery test** shows a larger subset does **not** raise the
  seal rate.
- Archive coverage (occupied niches / descriptor-space) is recorded pre/post steer, coverage collapse is visible,
  and a minimum-occupied-niches floor is enforced.
- Window/fold knobs cannot shrink the OOS span below the floor or drop the mandated stress regime (`400`); regime
  coverage is invariant to steering.
- The steer delta (subset hash + gens/pop + window/fold counts) is written into `VintageContent.lineage` via
  QE-467's schema; **no** second version bump is introduced.
- `validate_train` rejects any attempt to set a blocklisted threshold (cost-stress, turnover, capacity,
  DSR/PBO, IC/FDR, holdout/embargo/purge) below its compiled floor.
- The gate-monotonicity sweep passes: no whitelisted knob value moves a candidate from G1-reject to G1-seal.
- Including an evolved pool references only an **already-sealed** pool; no code path re-deflates or un-seals it,
  and the firewall/dependency-topology test stays green over the evolved-pool load path.
