# QE-464 — `ingest` run-kind + `POST /api/ingest` trigger + real-vs-synthetic provenance  *(R3 — real ingest)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#82-the-trigger--provenance-qe-464--run-kind--endpoint) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R3)` · `Area: server / storage / ingest` · `Depends on: QE-463, QE-467, QE-257, QE-440`

**Why.** With a real decoder (QE-463), the operator needs a supervised way to trigger ingest and — critically —
a way to tell **real from synthetic** data. Today `CoverageRow` (`crates/storage/src/coverage.rs:22`) carries
`{symbol, resolution, from, to, bars}` with **no source marker**, and `--synthetic` only warns at the CLI
(`crates/cli/src/jobs/ingest.rs:115`). Provenance must live in the store so nobody trains on synthetic bars
believing they are real.

**Scope / requirements.**
- An **`ingest` run-kind** (`RunSpec::Ingest(IngestParams)`, `crates/server/src/runs/model.rs:50`) +
  **`POST /api/ingest`** trigger: instruments, date range, resolution, and a **"fetch all available
  instruments"** option. Supervised like other runs (run store, subprocess, terminal `done` line — the `ingest`
  `done`-line writer already exists, `crates/run-protocol/src/lib.rs:212`); `validate_ingest` enforces required
  fields with a uniform `400`.
- **Fetch-all routes through the point-in-time universe machinery (kills survivorship).** Resolve "fetch all"
  through the **existing** as-of universe machinery — `crates/config/src/universe.rs`
  `InstrumentListing{listed,delisted}` + `crates/ingest/src/plan.rs::overlaps()` — mapping the exchange
  listing/delisting dates onto listing windows, and write the **resolved as-of instrument set** into
  coverage/lineage. AC: an **as-of-date backtest excludes not-yet-listed / already-delisted instruments**
  (survivorship-bias kill, QE-448). **If listing dates are unavailable in v1, fetch-all is flagged
  `survivorship-unsafe`** — not silently treated as an open-ended universe.
- **Real-vs-synthetic provenance (the headline requirement)** — thread a **`data_provenance` tag (`real` |
  `synthetic` | `mixed`)** into `VintageContent.lineage` **via QE-467's schema** (this ticket populates it; the
  field is defined by QE-467) and record the per-bar source on the store + coverage so the store itself knows
  every bar's origin. Extend `CoverageRow` (and the underlying store/coverage query) with the source tag; the
  `--synthetic` path writes `synthetic`, the QE-463 real path writes `real`. **Guard:** a synthetic/mixed input
  store must yield a vintage **marked** synthetic-/mixed-derived — never silently "real". Provenance is stored
  **key-scannably** so **coverage stays key-only** (QE-412); **interleaved real + synthetic is allowed but always
  labelled** — mixed coverage is written as **multiple contiguous rows, one per provenance run**, never a single
  blended row.
- **Liquidity screen (no capacity mirage on thin alts).** Capacity-eligibility requires **per-instrument rolling
  ADV / impact calibration (QE-440)** — the major-calibrated `$250k` floor is a mirage on thin alts. Flag or
  exclude thin names (pair with the `%ADV` participation guard, QE-447) so fetch-all does not silently admit
  illiquid instruments as tradable at size.
- **No `PROTOCOL_VERSION` bump** — the `ingest` `done`-line already exists in the protocol; and **no
  `input_snapshot_id` drift** for already-ingested data (re-tagging provenance must not change the snapshot id of
  bars already present).

**Out of scope.** The decoder itself (QE-463). The SPA trigger screen + provenance column (QE-465). Defining the
`data_provenance` lineage field or bumping any schema version (that is QE-467). Migration of pre-existing
untagged bars beyond a documented default (record them as unknown/legacy rather than guessing). Live/streaming
ingest.

**Acceptance criteria.**
- `POST /api/ingest` launches a supervised ingest run (instruments / date range / resolution, or fetch-all);
  invalid requests `400` uniformly; the run terminates with the standard `done` line; **no `PROTOCOL_VERSION`
  bump** is introduced.
- Fetch-all resolves through the as-of universe machinery (`InstrumentListing` + `plan::overlaps()`), writes the
  resolved as-of set into coverage/lineage, and an as-of-date backtest **excludes not-yet-listed / already-
  delisted** instruments; if listing dates are unavailable, fetch-all is flagged `survivorship-unsafe`.
- Every stored bar carries a `real`/`synthetic` provenance tag stored key-scannably (coverage stays key-only,
  QE-412); `data_provenance` is threaded into `VintageContent.lineage` (QE-467); a synthetic/mixed input store
  yields a vintage marked synthetic-/mixed-derived, never silently `real`.
- `GET /api/market-data/coverage` (QE-257) rows expose provenance; a store mixing real + synthetic reports the
  split as multiple contiguous per-provenance rows, never one blended row.
- The liquidity screen requires per-instrument rolling-ADV/impact calibration (QE-440) for capacity-eligibility;
  thin names (below the `%ADV` guard, QE-447) are flagged/excluded rather than admitted at the major floor.
- The `--synthetic` path tags its bars `synthetic`; a real-ingest run tags `real`; no path writes an untagged
  bar going forward; re-tagging causes no `input_snapshot_id` drift for already-ingested data.
