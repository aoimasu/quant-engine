# QE-464 ‚Äî `ingest` run-kind + `POST /api/ingest` trigger + real-vs-synthetic provenance  *(R3 ‚Äî real ingest)*

> **Status:** Todo ‚Äî [design QE-455](../../architecture/qe-455-research-flow-design.md#82-the-trigger--provenance-qe-464--run-kind--endpoint) ¬∑ [backlog index](../../backlog.md#research-flow) ¬∑ [spec](../../specs.md)

`Phase: Research-flow (R3)` ¬∑ `Area: server / storage / ingest` ¬∑ `Depends on: QE-463, QE-257`

**Why.** With a real decoder (QE-463), the operator needs a supervised way to trigger ingest and ‚Äî critically ‚Äî
a way to tell **real from synthetic** data. Today `CoverageRow` (`crates/storage/src/coverage.rs:22`) carries
`{symbol, resolution, from, to, bars}` with **no source marker**, and `--synthetic` only warns at the CLI
(`crates/cli/src/jobs/ingest.rs:115`). Provenance must live in the store so nobody trains on synthetic bars
believing they are real.

**Scope / requirements.**
- An **`ingest` run-kind** (`RunSpec::Ingest(IngestParams)`, `crates/server/src/runs/model.rs:50`) +
  **`POST /api/ingest`** trigger: instruments, date range, resolution, and a **"fetch all available
  instruments"** option resolved from the exchange's instrument-list endpoint. Supervised like other runs (run
  store, subprocess, terminal `done` line ‚Äî the `ingest` `done`-line writer already exists,
  `crates/run-protocol/src/lib.rs:212`); `validate_ingest` enforces required fields with a uniform `400`.
- **Real-vs-synthetic provenance (the headline requirement)** ‚Äî record a **provenance tag (`real` |
  `synthetic`)** on the stored data + coverage so the store itself knows every bar's origin. Extend
  `CoverageRow` (and the underlying store/coverage query) with the source tag; the `--synthetic` path writes
  `synthetic`, the QE-463 real path writes `real`. **Interleaved real + synthetic is allowed by design but must
  always be labelled** ‚Äî mixed-provenance coverage is surfaced explicitly, never silently blended.

**Out of scope.** The decoder itself (QE-463). The SPA trigger screen + provenance column (QE-465). Migration of
pre-existing untagged bars beyond a documented default (record them as unknown/legacy rather than guessing).
Live/streaming ingest.

**Acceptance criteria.**
- `POST /api/ingest` launches a supervised ingest run (instruments / date range / resolution, or fetch-all via
  the instrument-list endpoint); invalid requests `400` uniformly; the run terminates with the standard `done`
  line.
- Every stored bar carries a `real`/`synthetic` provenance tag; `GET /api/market-data/coverage` (QE-257) rows
  expose it; a store mixing real + synthetic reports the split explicitly rather than one blended row.
- The `--synthetic` path tags its bars `synthetic`; a real-ingest run tags `real`; no path writes an untagged
  bar going forward.
