# QE-460 — `RunSpec::Flow` composite run-kind + frozen-holdout carve  *(R2 — composite flow)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#5-the-composite-runspecflow-lifecycle) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R2)` · `Area: run-protocol / server` · `Depends on: QE-467, QE-452, QE-419, QE-458, QE-117, QE-125`

**Why.** train and backtest are two disconnected runs — `RunSpec` is a flat `Backtest | Train | Evolve`
(`crates/server/src/runs/model.rs:50`) and an operator trains, waits, copies the sealed vintage id into a
*second* form, and backtests it. There is no atomicity, no shared seed/holdout, no single lifecycle. This
ticket adds the **server-owned composite** `RunSpec::Flow`: configure once → `train`→`backtest` in one
supervised run with a deterministic vintage handoff and a **frozen OOS holdout carved once and recorded.**

**Scope / requirements.**
- Add **`RunSpec::Flow(FlowParams)`** to the enum (`model.rs:50`) with a `FlowParams` DTO in the
  dependency-free `qe-run-protocol` leaf crate (bump `PROTOCOL_VERSION 2→3`; every field `#[serde(default)]`
  **except** the required window + a **required `seed`**, mirroring `EvolveParams.seed`,
  `crates/run-protocol/src/lib.rs:446`). `FlowParams` embeds the QE-458 steer-whitelisted train block + the
  backtest window. `run_type()=="flow"`, `writes_vintage()==true` (`model.rs:96` gains the `Flow` arm),
  `label()`=the flow window.
- **`validate_flow`** in `crates/server/src/runs/manager.rs` (beside `validate_train`/`validate_evolve`):
  required window + seed; QE-458 steer-whitelist enforcement; holdout/embargo **floors** (design §4); uniform
  `400`. Reuse the QE-419 single-source-of-truth config pin for the storage dirs the sub-jobs share.
- **Server-owned sequencing** — run the `train` sub-job to a sealed vintage, then spawn the `backtest` sub-job
  over the **frozen holdout window** with the just-sealed vintage id (a **deterministic, content-hash vintage
  handoff**). The flow succeeds only if both sub-runs succeed and the vintage sealed; a train that fails G1
  fails the flow and runs no backtest. One run-store row, one status, sub-run ids in `meta.json`.
- **(a) Collapse the holdout contract — one consultation, no double credit.** The search/training folds are
  disjoint + embargoed from the holdout; the flow's backtest is the **single recorded consultation** of the
  holdout and confers **NO independent deflation credit** — the flow-backtest number **IS the gate's holdout
  evaluation re-surfaced**, not a fresh OOS sample. **Delete** the earlier contradictory "backtest window
  disjoint from holdout" language: the backtest evaluates *on* the holdout (it is the holdout consultation), it
  is not a second disjoint OOS window.
- **(b) Holdout geometry — regime-stratified / walk-forward, not a single trailing block.** Carve the holdout as
  **regime-stratified or multi-fold walk-forward**, reusing **QE-117** WFO windows + **QE-125** regime labels;
  assert **≥ K regime labels** or **N non-contiguous embargoed folds** are covered; record the **regime
  composition** in lineage (via QE-467). If a single trailing holdout is retained for v1, its window edge is
  **server-derived from the pinned input snapshot's right edge**, not operator-chosen.
- **(c) Consultation counter — actually increment + record, overlap-keyed.** Increment and record a per-holdout
  **consultation count** (into QE-467's lineage field), **keyed on holdout-bar OVERLAP** — any sealed vintage
  whose holdout **intersects** this run's, or whose training window covers a prior holdout — **not** exact range
  equality. This makes campaign-level holdout re-use visible even when windows only partly coincide.
- **(d) Pin the same sealed cost calibration across sub-runs.** The train and backtest sub-runs use the **same
  sealed cost calibration** (assert cost-model equality — maxdama #6 parity), so the backtest cannot re-cost the
  holdout under a friendlier friction model than the gate used.
- **Frozen OOS holdout carve** — compute the holdout split (bars reserved + embargo) **once, before search**;
  hand the *same* split to both sub-runs. Record `{holdout_range, embargo, train_range}` + regime composition in
  `VintageContent.lineage` (via QE-467's schema, design §4) so the verdict's data is reproducible & auditable.
- **(e) `PROTOCOL_VERSION` 2→3** for the new run-kind, updating the `assert_eq!(PROTOCOL_VERSION, 2)` at
  `crates/run-protocol/src/lib.rs:31`. **(f) The lineage fields this flow writes ride QE-467's single
  `VINTAGE_FORMAT_VERSION` 7→8 bump — do NOT bump the vintage format again here.**
- **Determinism** — the single flow `seed` derives the train search seed and the backtest; re-running from
  `seed` + pinned `input_snapshot_id` reproduces the vintage byte-identically (QE-006/QE-129 content-hash).

**Out of scope.** Resume/checkpoint/halt + the concurrency lane (QE-461). The SPA Flow page (QE-462). Any
change to `evaluate_g1`, the search, the ensemble DE, or the seal. Any promote/select across flows (§9).
Defining the lineage/regime/consultation schema or bumping `VINTAGE_FORMAT_VERSION` (that is QE-467 — this flow
only *populates* those fields).

**Acceptance criteria.**
- A `type:"flow"` create → one supervised run that sequences train→backtest atomically with a content-hash
  vintage handoff; a failed-G1 train seals nothing and runs no backtest; the flow row carries one status.
- The backtest is the **single recorded holdout consultation** conferring no independent deflation credit; the
  flow-backtest number is the gate's holdout evaluation re-surfaced (no "backtest disjoint from holdout" claim
  remains).
- The holdout is **regime-stratified / multi-fold walk-forward** (≥ K regime labels or N non-contiguous
  embargoed folds), floored (a request shrinking it below the floor or zeroing embargo `400`s), embargoed from
  folds, with its edge server-derived from the pinned snapshot if a single trailing block is used; the split +
  regime composition are written into `VintageContent.lineage` (QE-467 schema).
- The consultation counter increments and records **overlap-keyed** (holdout intersection, not exact equality).
- The train and backtest sub-runs pin the **same sealed cost calibration** (asserted equal — maxdama #6 parity).
- Re-running a flow from its recorded `seed` + pinned snapshot reproduces the vintage byte-identically.
- `PROTOCOL_VERSION` is bumped to 3 (assertion updated); **no** second `VINTAGE_FORMAT_VERSION` bump is added
  (the lineage fields ride QE-467's bump); the firewall/dependency-topology test stays green (no new cross-crate
  edge — the supervisor sequences existing CLI sub-jobs).
