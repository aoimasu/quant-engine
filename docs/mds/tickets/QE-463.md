# QE-463 — Real `http` ingest decoder for one exchange (Binance USDT-M)  *(R3 — real ingest, the long pole)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#81-the-decoder-qe-463--one-exchange-behind-http) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R3)` · `Area: ingest` · `Depends on: QE-253`

**Why.** Training data is synthetic-only: `qe ingest --synthetic` (`crates/cli/src/jobs/ingest.rs:107`) was
merged, but the real `http` decoders are a documented future-work seam — `HttpFetcher` over `ureq` compiled
only under the `http` feature (`crates/ingest/src/fetcher.rs:35`, `Cargo.toml` `http = ["dep:ureq"]`) and
`run_ingest` written against the injectable `HistoricalSource` (`crates/cli/src/jobs/ingest.rs:50`). Fill in
**one** exchange decoder so a flow can train on real data. This is honestly the **long pole** (design §8.4):
rate limits, pagination, funding/kline cadence, API drift, and data-licence reality — hence a deliberately thin
slice.

**Scope / requirements.**
- A **Binance USDT-M** historical decoder (klines + the historical funding series) behind the default-off
  `http` feature, plugged into the existing `Fetcher`/`HistoricalSource` seams and reusing the QE-101..103
  planner/reconciliation where applicable.
- **Incremental** — fetch only the periods missing vs current store coverage (`coverage_bounds`,
  `crates/storage/src/coverage.rs`); do not re-download covered ranges.
- **Resumable** — an interrupted download continues from where coverage ends, not from the start.
- **Idempotent** — re-fetching a covered period is a no-op producing byte-identical bars, so retries are safe.
- **Default-off / offline CI** — all real-network code stays behind `#[cfg(feature = "http")]`; the default
  build, the `--synthetic` path, and the in-memory test sources are untouched; CI stays offline.

**Out of scope.** The run-kind + trigger endpoint + provenance marker (QE-464). The SPA (QE-465). Multi-exchange
/ multi-asset-class ingest. Live/streaming ingest (that is the runtime side, QE-202..205). Any change to the
`--synthetic` generator.

**Acceptance criteria.**
- Under `--features http`, the decoder fetches Binance USDT-M klines + funding for a small instrument set over a
  date range, incrementally (only missing periods), resumably (continues after interruption), and idempotently
  (re-fetch = byte-identical no-op).
- With the feature off, the crate/CLI build and existing offline tests are unchanged; no real network call is
  reachable in the default build.
- Decoding logic is exercised offline against a fixture (no live network in the test).
