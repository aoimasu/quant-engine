# QE-457 — SPA Vintage Inspector screen (mirrors evolve PoolReview)  *(R1 — inspection)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#72-the-vintage-inspector-screen-qe-457--mirrors-poolreview) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R1)` · `Area: frontend` · `Depends on: QE-456, QE-467, QE-259`

**Why.** The SPA still renders a placeholder for the vintage/strategies browser (*"The strategies browser is on
the way"*, `web/src/app/App.tsx:121`), while `evolve` has a full inspection screen
(`web/src/app/evolve/PoolReview.tsx`). Now that QE-467 **persists** the full evidence and QE-456 exposes it, the
inspector must **surface those persisted details as first-class UI** — an operator must never have to leave the
screen to learn a vintage's provenance, deflation basis, or holdout regime coverage.

**Scope / requirements.**
- A Vintage Inspector screen consuming `GET /api/vintages/{id}` (QE-456), reachable from the vintages list
  (replacing the `App.tsx:121` placeholder), structured like `PoolReview.tsx`:
  - **Provenance banner (first-class, prominent).** A `data_provenance` banner that makes a **synthetic- or
    mixed-derived vintage UNMISTAKABLE** — mirroring the CLI `"synthetic":true` honesty — so no operator reads a
    synthetic-derived vintage as real. `mixed` is called out distinctly, never softened to `real`.
  - **Composition table** — chromosome → referenced indicators (catalogue vs evolved, labelled) → weight.
  - **Selected-indicators panel** — the union of indicators the ensemble references.
  - **Gate-evidence card** — leads with the **net-of-cost / tradability** numbers (cost-stress `min{1×,2×}`,
    **realised turnover, `capacity_usd`**) and the **full honest deflation basis** (DSR labelled
    "necessary — not sufficient", **uncensored PBO with its population**, distinct-trial **N** vs the
    `E[maxSharpe]` bar, **IC/FDR**) — the same hierarchy QE-450 §13.5 mandates for PoolReview. Sharpe/DSR
    demoted, never a lone green tile. All numbers are the QE-467-persisted values rendered verbatim.
  - **Frozen-holdout panel** — the recorded `{train_range, embargo, holdout_range}` split **and the holdout
    regime composition** (which regime labels the holdout spans, QE-125) so the operator sees the verdict rode
    diverse regimes, not one trailing block.
  - **"Backtest-holdout only — not paper-confirmed" label** — a standing label stating the verdict is a
    backtest-holdout evaluation and still owes the G2/G3 live/shadow gates before promotion.
- Read-only: **no seal / promote / select affordance** (the vintage is already sealed by the train gate).

**Out of scope.** The endpoint (QE-456) and the persistence (QE-467). The leaderboard/comparison view (QE-466).
Steering controls (QE-459). Any client-side gate recomputation (render server-provided numbers verbatim).

**Acceptance criteria.**
- The screen renders the provenance banner, composition (chromosome→indicators→weight), selected indicators, the
  full gate/deflation evidence, and the holdout split + regime composition from QE-456's payload; the placeholder
  is removed.
- A synthetic- or mixed-derived vintage shows an unmistakable provenance banner; a `mixed` vintage is not shown
  as `real`.
- The gate card leads with net-of-cost/tradability (cost-stress, turnover, `capacity_usd`) and shows the
  deflation basis honestly (DSR necessary-not-sufficient, uncensored PBO + population, distinct-trial N vs bar,
  IC/FDR); it presents no standalone health badge, and carries the "not paper-confirmed" label.
- No promote/seal/select control exists anywhere on the screen (inspection only).
- Component tests cover the composition table, the gate card, and the provenance banner (real/synthetic/mixed),
  mirroring `PoolReview.test.tsx`.
