# QE-459 — SPA steering controls on the New-training / flow form  *(R1 — steering)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#6-the-steer-knob-whitelist--the-anti-overfitting-guardrail) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R1)` · `Area: frontend` · `Depends on: QE-458, QE-261`

**Why.** QE-458 makes the search steerable server-side; the operator needs the controls to drive it. The
Training UI (`web/src/app/training/`, QE-261) currently exposes only a subset of `TrainParams`. Surface the
whitelisted knobs — and, mirroring the `evolve` NewCampaign guardrail chips (QE-450 §13.4), make the
**blocklisted thresholds visibly fixed** so the UI teaches the boundary rather than inviting a bypass.

**Scope / requirements.**
- Add steering controls reflecting QE-458's **whitelist** to the New-training form (and reused by the Flow page
  QE-462):
  - **Indicator picker** — select catalogue indicators + include/exclude already-sealed evolved-pool formulas.
  - **Search-budget** inputs — generations / population.
  - **Windows/folds** inputs — number & length.
- Render the **blocklisted** thresholds (cost-stress multiplier, turnover cap, capacity floor, DSR/PBO cutoffs,
  holdout/embargo floors) as **fixed, disabled guardrail chips** with a hint that they are compiled floors the
  research path cannot relax — mirroring `evolve` NewCampaign's disabled caps chips.
- Client-side hints only *remove* affordances; the server (`validate_train`) remains the enforcement point.

**Out of scope.** The server validation/whitelist (QE-458). The composite Flow page layout (QE-462). Any control
that could submit a blocklisted-threshold override (the field does not exist in the form).

**Acceptance criteria.**
- The form exposes the indicator picker, budget, and window/fold controls, and submits them as QE-458's
  whitelisted `TrainParams` fields.
- Blocklisted thresholds render as disabled guardrail chips; there is no form control that can set them.
- Component tests cover the indicator picker and the disabled guardrail chips.

`Spec ref: QE-455 §6 (whitelist/blocklist); QE-450 §13.4 (guardrail chips — client ergonomics, server enforces).`
