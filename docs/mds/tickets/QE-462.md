# QE-462 — SPA single stepped Flow page (configure train+backtest+steer once → run)  *(R2 — composite flow)*

> **Status:** Todo — [design QE-455](../../architecture/qe-455-research-flow-design.md#5-the-composite-runspecflow-lifecycle) · [backlog index](../../backlog.md#research-flow) · [spec](../../specs.md)

`Phase: Research-flow (R2)` · `Area: frontend` · `Depends on: QE-460, QE-459`

**Why.** The composite `RunSpec::Flow` (QE-460) exists server-side; the operator needs a single page to
configure the whole train→backtest+steer flow **once** and launch it — instead of the current two-form,
copy-the-vintage-id dance.

**Scope / requirements.**
- A single stepped **Flow** page that composes, in one form: the QE-459 steering controls (indicator picker,
  budget, windows) + the training window + the backtest window + the frozen-holdout size/embargo (shown with
  their compiled floors as disabled guardrail chips, per QE-459/§4) → one `POST /api/runs` with `type:"flow"`.
- Reuse the existing training/backtest form components (`web/src/app/training/`, `web/src/app/backtest/`) rather
  than rebuilding them; a stepped layout (configure → review → launch) over the single `FlowParams` body.
- After launch, open the flow's monitor (one run row, one status), surfacing the **per-phase progress**
  (train phase → backtest phase) and, on success, a link to the sealed vintage in the Inspector (QE-457).
- **Surface the persisted holdout details (inherits QE-467/456 data).** Show the **holdout / regime chips**
  (which regimes the frozen holdout spans, QE-460/467) and the **"backtest-holdout only — not paper-confirmed"**
  label on the flow result, mirroring the Inspector (QE-457) — so the operator reads the flow verdict as a
  backtest-holdout evaluation still owing G2/G3, not a live-confirmed result.

**Out of scope.** The server composite/supervision (QE-460/461). The steering controls themselves (QE-459 —
reused here). A dedicated `web/src/app/flow/` area (design §14 open question; a stepped page reusing existing
components is the lean choice unless the flow list grows its own lifecycle).

**Acceptance criteria.**
- The Flow page configures steer + train window + backtest window + holdout in one form and submits a single
  `type:"flow"` create; holdout/blocklist thresholds show as disabled guardrail chips.
- The monitor shows one run row with the train→backtest per-phase progression, the holdout/regime chips, the
  "not-paper-confirmed" label, and links to the sealed vintage's Inspector on success.
- Component tests cover the stepped form, the single-`POST` submission shape, and the holdout/regime +
  not-paper-confirmed rendering.

`Spec ref: QE-455 §5 (composite flow); §14 Q6 (stepped page vs dedicated area).`
