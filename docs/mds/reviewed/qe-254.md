### Goal
The admin UI needs a backend. A new `qe-server` crate is a second composition root (ADR D4a): all-Rust,
async isolated to this crate, reusing the engine crates, forbidden from `qe-runtime`/`qe-venue`.

### Acceptance criteria (from backlog)
- [x] `qe-server` builds and serves a health endpoint + a static index.
- [x] The firewall/decoupling tests pass and now cover `qe-server` (no `qe-runtime`/`qe-venue` edge).

### Implementation summary (scaffold only)
- New `crates/server` (`qe-server`) binary; added to root `[workspace.dependencies]`. axum + tokio + tower-http.
- `GET /api/health` → `200 {"status":"ok"}`; static SPA at `/` via `tower-http` `ServeDir` + `index.html`
  fallback (graceful 404 if absent); `/api` reserved via nested sub-router with own 404 fallback (unknown
  `/api/*` NOT swallowed by SPA fallback). 12-factor `ServerConfig::from_env`: `QE_SERVER_ADDR`,
  `QE_SERVER_STATIC_DIR` (CWD-relative default).
- Firewall rule `qe-server ⊥ {qe-runtime, qe-venue}` (`architecture/src/lib.rs`) + firewall-test anti-vacuity
  coverage; `cargo metadata` transitive-closure decoupling assertions (`cli/tests/dependency_topology.rs`).
- Async integration tests (`server/tests/http.rs`, tokio + `tower::oneshot`): health JSON, static index, SPA
  deep-link fallback, reserved-`/api` 404, and graceful-404-when-static-absent (pass-2).
- **deny.toml: NONE** (axum/tokio/tower-http tree all MIT/Apache-2.0/ISC/BSD/Unicode-3.0/Zlib, already allowed).
