## Decision summary

`LiveKlineSource` (qe-runtime) reconstructs live multi-resolution bars by streaming. QE-106's
`BarReconstructor` is already streaming-shaped (`reconstruct_batch` == `new` + push-each + finish), so the
source reuses it unchanged, one per target tier, and adds only the **stitch** — a monotonic open-time dedup
(`open_time <= last_open_ms` → dropped) over the REST-prime / wss-continuation boundary. Per-tier output is
therefore `reconstruct_batch(deduped_base, M5, tier)` bar-for-bar — parity by construction, not coincidence.

## Review (pass 1) — Approved

Principal-level pass against the AC. Verdict: **Approved.** No blocking issues, no non-blocking notes
needing action.

- **(1) Parity structural + test non-vacuous** — `reconstruct_batch` is exactly the same code path; the

- **(2) Stitch `<=` boundary correct** — open_time is the bar's window identity; an equal-time bar is a

- **(3) Fan-out/order** — one reconstructor per tier, tier-order within a step / time-order across; 48 bars
  → 8×M30 + 1×H4.
- **(4) Delegation** — wrong base resolution → `UnexpectedResolution` (checked before the stitch); `?`
  propagates all reconstructor errors, nothing swallowed.

No outstanding comments. The reuse-don't-reimplement design makes parity a structural guarantee. Approved &
