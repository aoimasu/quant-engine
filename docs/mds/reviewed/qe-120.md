## Decision summary

Integration of QE-110 `decide` + QE-109 `friction` + QE-113 `fitness`. The signal at bar `i` fills at
`i+1`'s price (no look-ahead). A `Decimal` cash/mark ledger has fees/slippage/funding reduce cash, so
every `r_i = equity_i/equity_{i-1} − 1` is net-of-cost. Sizing `(size_bps/10000)·equity` (≤ 1×
leverage). The net return series is split into `windows` contiguous sub-windows → `NoiseRobustFitness`
mean ± SE. `trades < min_trades` ⇒ rejected as noise (`fitness.mean = −∞`, never an elite). The result
feeds QE-113 `should_replace` (SE-aware). Scalar archive fitness = `fitness.mean`.

## Review (pass 1) — Approved

Principal-level review of `494aff4`. Both ACs genuinely met — net-of-cost by construction, no look-ahead,
trade gate and SE-aware replacement work; the `Decimal` ledger and ruin path traced by hand. No real bugs.

- **(a)** `apply_fill` subtracts the notional exchange (equity-neutral at mark) then `(fee+slip)·mult`;
  funding subtracts `pos_qty·price·rate` — all reduce equity ⇒ every `r_i` net-of-cost. The
  cost-multiplier test is a real proof: `decide` is cost-blind so the trade *sequence* is identical
  across cheap/dear runs (now also asserted `lo.trades == hi.trades`), isolating cost drag.
- **(b)** Order from `decision_{i-1}` fills at `price_i`; `decision_i` computed after marking `equity_i`,
  fills at `i+1`. No self-fill; one-bar series books 0 trades.
- **(c)** `trades` counts entry fills only; `<min_trades` ⇒ `mean=−∞`, `should_replace` rejects it.
- **(d)** Real `n=4`, `std_error>0`; `+0.5·SE` rejected, `+5·SE` replaces; `split_windows` partitions
  contiguously, sums to `len`, sizes differ by ≤1.
- **(e)** ≤1× sizing off `equity_prev`, `price>0`-guarded division, exact flat close, correct funding
  sign, `equity_prev==0` guard; ruin path (full leverage + price→0.01 ⇒ `r<−1` ⇒ `−∞`) traced. No panic.
