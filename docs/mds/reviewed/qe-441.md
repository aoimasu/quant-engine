## Review verdict — [Approved] (0 blocking, 3 non-blocking)
1. **Injection point correct.** `shock_loss = |pos_qty·price| · adverse_fraction` deducted from cash **before** equity is marked; since `|pos_qty·price| = size_frac·equity_prev`, loss scales **linearly with size** → larger `size_bps` takes a strictly deeper shocked drawdown (→ ruin past a threshold). Both rolls consumed every bar → schedule is a pure fn of (seed, bar) — position-independent. Genuine direction tests (deeper-dd-at-larger-size; shocked optimum strictly below the no-shock top-of-sweep; monotone non-increasing in severity).
2. **Seeded AND frozen — both hold.** Seeded: per-call `seed_rng(cfg.shocks.seed)`, `backtest` pure fn of (genome,bars,cfg), thread-count independent. Frozen: `ShockConfig::default()` uses a fixed `DEFAULT_SHOCK_SEED` (NOT `params.seed`) + frozen magnitudes with no override path → severity/frequency not a per-run DOF (Math#2 addressed); content-addressed + sealed as a hashed `VintageContent.shocks` field.
3. **Scope containment coherent — deflation on HONEST returns.** Only `search_cfg` is shocked (→ selection fitness). Elite pool (ensemble/DSR/SPA), holdout/G1, Kelly sizer, `cell_champion_returns` (DSR variance), and reporting/replay all run `shocks: None`. So DSR/PBO/G1 (V, N, candidate) is entirely honest — shock severity can't distort the deflation basis; the shock is a sizing-prudence regularizer on selection, not a deflated/reported statistic. Empirically confirmed (golden metrics/trades byte-unchanged).
4. **Firewall clean** (`qe-wfo` has zero `qe-ensemble` deps; `ShockConfig` from the `qe-risk` leaf; shapes replicated). **Golden reproduced** byte-for-byte (`1dbb7240…`, field diff only `shocks` + format bump).

### Non-blocking follow-ups (accepted)
