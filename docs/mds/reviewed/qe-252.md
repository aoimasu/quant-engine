### Goal
`BacktestResult` emitted per-bar returns and a trade **count** only. The admin UI's Trades tab + the
win-rate / profit-factor / Sortino metrics need per-trade data. Close the gap without disturbing the
existing hot-path result.

### Acceptance criteria (from backlog)
- [x] A known single winning round-trip yields exactly one `TradeFill` with `return_frac > 0`.
- [x] The existing `qe-wfo` suite still passes unchanged.
- [~] `win_rate`/`profit_factor` match hand-computed values — **deferred to QE-251** (`crates/cli/src/jobs/metrics.rs`)
  per the v1 plan; QE-252 delivers only the `qe-wfo` trade-recording substrate. Reviewer confirmed the split is sound.

### Implementation summary
- `qe_wfo::backtest::backtest_with_trades(genome, bars, cfg) -> (BacktestResult, Vec<TradeFill>)`;
  `backtest()` delegates via `.0` (hot-path result byte-identical by construction — recorder only touches
  new `open`/`fills` locals, never the ledger/returns/fitness).
- `TradeFill { entry_idx, exit_idx, side: Direction, entry_px: Decimal, exit_px: Decimal, return_frac: f64 }`,
  one per closed round-trip; `open = Some((i, dir, price))` set inside the entry guard in lockstep with the
  `trades` counter, consumed via `open.take()` in the close arm. `return_frac` is signed gross price return
  (Long `(exit−entry)/entry`, Short `(entry−exit)/entry`).
- Exported `backtest_with_trades` + `TradeFill` from `qe_wfo::lib`.
- TDD tests: `single_winning_round_trip_records_one_trade`, `single_winning_short_round_trip_records_one_trade`
  (short-side sign coverage, added for review nit), `backtest_delegates_to_with_trades`,
  `open_position_at_end_records_no_trade`.
