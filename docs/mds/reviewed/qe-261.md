### Implementation summary
- **Part A (server):** `manager::create` dispatches request `type` into internal `RunSpec{Backtest|Train}`;
  `CliJobSpawner::spawn` builds the `backtest` argv byte-verbatim, or a `train --run-dir --json [--start/--end/
  --resolution/--seed/--generations/--population/--holdout/--embargo/--config/--profile]` argv. Supervisor stdout
  parser gained QE-260 `gen`/`ensemble`/`gate` + `done{vintage}`; each updates the coarse `meta.progress` (train
  reaches pct=100 on done, guarded to Train) + a lazily-created `RunMeta.train: Option<TrainProgress>`. Backtest
  `meta` unchanged (params stored as opaque `serde_json::Value`; `train` skip-if-none). Train float fields
  `Option<f64>` so a `best_fitness:-inf/NaN`‚Üínull line survives (tested). Train spawned as subprocess (firewall green).
- **Part B (frontend):** net-new `TrainingMonitor` ‚Äî trigger ‚Üí `POST /api/runs` `type:"train"`; resilient polling
  (QE-259 pattern) renders generation + best-so-far, MAP-Elites archive-coverage grid, CV folds/members/score, and
  the G1 verdict (promoted/failed criteria + in-sample/holdout Sharpe, DSR, SPA p-value, n_trials). On succeeded:
  G1 badge + "Backtest this vintage" ‚Üí one-shot deep-link preselecting the sealed vintage in the QE-259
  New-backtest flow. Added to Research nav; CSP-safe.
