## Decision summary

`LiveFactorJoin` (qe-runtime) produces factor rows on the live path: an as-of join of live scalar context
(funding/open-interest/premium) onto base bars, driving the **unmodified** shared `FeatureAssembler::push`.
Context observations update a last-known snapshot (streaming as-of over time-ordered events); `on_bar`
pairs the snapshot in force with the bar (= most-recent context with `ts <= bar.open_time`) into a
`qe_signal::Sample`. So a live factor row equals `assemble_batch`'s vector for the same sample by
construction; only the join is new.
