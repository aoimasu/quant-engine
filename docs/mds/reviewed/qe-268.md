### Goal
Turn `qe-error`'s documented "order-emission modules must reject unwrap/expect/panic" rule — enforced today only on the `error::hot_path` demonstrator — into an actual compiler-enforced invariant on the seven live runtime order-path modules, converting every production `.expect()` there to a total/proof-carrying form.

### Acceptance criteria (from `docs/backlog.md` QE-268)
- [x] Every listed order-emission module (`edge`, `kill_gate`, `pretrade`, `hedger`, `live_netter`, `live_breakers`, `evaluator`) carries `#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic)]`.
- [x] `cargo clippy --workspace --all-targets` is green.
- [x] Introducing an `unwrap`/`expect`/`panic!` into any of those modules fails clippy (prove-it).
- [x] Live behaviour unchanged — `order_port_conformance` + `restart_parity` green.

### Reviewer sign-off (principal review, 2026-07-04)

**Verdict: [Approved] — meets all acceptance criteria, no blocking issues. 0 blocking, 0 non-blocking comments.**

New API surface — verified correct and behaviour-preserving against source:
- ✅ `Qty::abs_of(v) = Qty(v.abs())` (money.rs:126) — non-negative by construction. `Qty::new` (money.rs:109) only validates `value >= 0` then wraps with no rounding, so `Qty::new(mag).expect()` ≡ `Qty(mag)`. At every site the magnitude fed to the old `new` was already `|signed|`, so `abs_of(signed)` yields the bit-identical `Qty(|signed|)`:
  - #2 `plan_delta`/#5 `flatten_intent`: old `mag` = `-x`/`x` = `|x|`; `side` still branches on `is_sign_negative()` (mapping preserved). Zero → positive branch, `abs_of(0)=Qty::ZERO`, unchanged.
  - #3/#4 `position_report` (edge.rs:343-349): neg branch `-signed_qty = |signed_qty|`, pos branch `signed_qty = |signed_qty|`; pre-existing `is_zero()` guard unchanged.
- ✅ `impl Add for Qty = Qty(self.0 + rhs.0)` (money.rs:142) — sum of two non-negative `Qty` is non-negative; equals old cumulative-fill `expect` at site #1. Overflow caveat matches existing `Notional` `Add`; far outside realistic fill magnitudes.
- ✅ `Fraction::ZERO`/`ONE = Fraction(Decimal::ZERO/ONE)` (limit.rs:116/119). `Fraction::new` validates `[0,1]` then wraps; `0`/`1` in range, so consts exactly equal old `Fraction::new(0/1).expect()` at sites #6/#7. Correctly placed in `qe_risk`.
