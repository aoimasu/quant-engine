## Decision summary

`label_regimes(bars, cfg)` labels each bar on two deterministic axes — volatility (`Calm`/`Volatile`,
median split of rolling realised vol) and trend (`Trending`/`Choppy`, Kaufman efficiency ratio vs
`trend_threshold`); first `window` bars `None`. `expectancy_table(returns, labels)` pairs a per-bar return
series with the labels and aggregates per regime (count/mean/total/win-rate) + `unlabelled`. Lives in
`qe-signal` (the only crate both `qe-wfo` and `qe-ensemble` depend on — firewall-safe). No HMM/RNG →
deterministic. No new deps.

## Review (pass 1) — Approved

Principal-level review of `00ca7c6`. The expectancy table is correct and strategy-agnostic, the labeller
has no NaN/div-by-zero paths, placement respects the firewall, and the median-split test is an honest
choice. No real bugs.

- **(a)** `expectancy_table` pairs `returns[i]`/`labels[i]` by index over `n = min(len, len)`, accumulates
  into a `BTreeMap` (ascending `Regime` order), tallies `None` into `unlabelled`; `labelled() + unlabelled
  == n` is a structural invariant catching alignment miscounts. Strategy-agnostic (constant `[0.01;n]` →
  mean 0.01, win_rate 1 per row).
- **(b)** Placement in `qe-signal` is the only firewall-safe home; `qe_domain::Bar` is a valid downward
  import, no cycle.
- **(c)** Rolling vol (std of log-returns), Kaufman ER in [0,1], even/odd median, warm-up `None`; all
  NaN/div-by-zero paths guarded; `n ≤ w` early return correct.
- **(d)** Median-split fracture is inherent (not a bug); the test honestly asserts rate-ordering between
  segments; `v > median ⇒ Volatile` a defensible convention.
- **(e)** Pure, no RNG, total-order sort, `BTreeMap` ordering — deterministic.
- **(f)** Empty bars, `window=0` (→ `max(1)`), empty returns/labels, single regime all handled.

Two non-blocking forward notes, no action required: (1) the AC fixture's `trend > chop` is mildly
self-referential (the labeller is tested independently, so fairly covered); (2) the median split is a
documented config-ready baseline with a quantile/HMM upgrade tracked.

No outstanding comments. Approved & merged.
