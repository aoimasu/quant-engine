## Decision summary

Venue-aware REST client in `qe-venue`. Data path **client → rate-limit handler → ephemeral cache →
transport**. `RateLimiter` is a pure rolling-window weight budget that backs off (`Ready` /
`WaitUntil(roll_instant)`) and **never drops**; `note_retry_after` floors the next-allowed instant from a
venue `429/418`. `Clock` seam (`SystemClock` prod / `ManualClock` test) makes backoff deterministic.
`RestCache` memoises only immutable closed-window (`window.end() <= now`) responses, below the limiter (a
hit costs no weight/transport). `VenueRestClient` does cache read-through → rate-limited+retried transport
→ cache write-back, with `paginate`; bounded retries surface `RestError::Exhausted` rather than looping.
`HttpRestTransport` (ureq, system TLS) behind the default-off `http` feature.
