## Decision summary

The wss half of `qe-venue`. `StreamTier { Market, Realtime }` partitions a `ConnectionRegistry`
(`HashMap<StreamTier, TierConn>`); each tier owns an independent connection, recorded subscriptions, and
`last_event_ms`. `StreamChannel { Kline(Resolution), MarkPrice }` carries `cadence_ms()` and venue
`stream_name()`s. The `WsConnection`/`WsConnector` seam is pull-based (`poll() →
Message/Disconnected/Idle`) so the registry is single-threaded and deterministic. On `Disconnected`, `pump`
reconnects and resubscribes the recorded subscriptions on a fresh socket; `last_event_ms` survives the
swap, so the first resume message reports `Gap{from,to}` when `event_time − last > cadence`. The concrete
async adapter is deferred to runtime wiring (keeps offline build + `deny` light).

## Review (pass 1) — Approved

Principal-level pass against the AC. Verdict: **Approved.** No blocking issues.

- **(1) Partitioning real** — independent `conn`/`subs`/`last_event_ms` per tier; the test pins both
  non-bleed directions (Realtime sub absent from Market and vice-versa) with distinct connects.
- **(2) Reconnect + resubscribe** — `Disconnected` connects a fresh socket, reassigns `entry.conn`, then
  replays the recorded `subs` on the new socket (test: 2nd connect + 2nd subscribe).
- **(3) Gap crux** — `last_event_ms` survives the outage (only `conn` swapped); the gap surfaces on the
  first resume message (not the reconnect pump); `> cadence` strict is correct (one-cadence resume is
  contiguous → no gap, pinned); in-stream skip reported without a disconnect; first-ever message yields no
  spurious gap.
- **(4) Determinism** — pull-based `poll()`, scripted fakes, no threads/sockets/sleeps; concrete adapter
  deliberately deferred.
- **(5) Cadence/names** — M5→300k, H4→14.4M, markPrice→1000; `btcusdt@kline_5m`, `btcusdt@markPrice@1s`.
- **(6) Firewall** — no new deps; QE-132 guard green.

One trivial non-blocking note: a resubscribe failure after the socket swap leaves a
connected-but-unsubscribed slot (the `WsError` is surfaced, so the caller can re-pump — no silent loss);
out of scope for the AC, a later resilience pass could record a resubscribe-pending state. No action taken
(marginal edge on a seam whose concrete adapter isn't yet wired).

No outstanding comments. Approved & merged.
