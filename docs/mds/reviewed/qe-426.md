## Review verdict — [Approved]
1. **Pure move (core).** `git -M` shows almost every module a 100% rename; sub-100% files (`edge.rs` 96%, `shadow`/`transport`/`pretrade` 99%, `hedger.rs` 71%) change ONLY imports/doc-link paths. The three contract types (`CapitalView`, `PositionKeeper` incl. default `equity()`, `TargetPosition` + `single`/`direction`) and the `PositionKeeper for &K` blanket impl were relocated VERBATIM into `qe-runtime-core` (orphan rule forces the blanket impl to live with the trait). The QE-421 `classify.rs` split by type-ownership into private `mod classify` in edge (KillHalt/Transport/Append) and hedger (Bootstrap/Cutover/BootState), impl bodies IDENTICAL to PR #123. The taxonomy test moved to `crates/runtime/tests/classify_taxonomy.rs` (orphan rule) with assertions CHARACTER-IDENTICAL (only dedent + facade imports + a move-required `#[allow(dead_code)]`). The three existing integration tests are NOT in the diff (byte-identical) and pass unchanged — strongest behaviour-preservation evidence.
2. **Order path as its own crate.** `cargo build -p qe-edge` standalone; deps tight (domain/error/risk/venue + `qe-runtime-core`). `#![deny(clippy::unwrap_used, expect_used, panic)]` rides verbatim on `edge.rs`+`kill_gate.rs` (+ `qe-runtime-core`); `cargo clippy -p qe-edge` clean — order-submitting code isolated + panic-free-scoped.
3. **Firewall covers each split crate.** New rules match the AC exactly: `qe-edge ⊬ {wfo,ensemble,vintage,signal,hedger}`, `qe-hedger ⊬ {edge,wfo,ensemble}`, `qe-runtime-core ⊬ {edge,hedger,venue,risk,signal}`; existing wfo/ensemble/server forbidden lists extended to the three split crates. Tight for the edge security boundary. Non-vacuity guarded: the test asserts real edges parsed (`qe-edge→qe-venue`, `qe-hedger→qe-vintage`, `qe-runtime-core→qe-domain`, `qe-runtime→qe-edge`) so no new rule passes vacuously.
4. **gRPC/transport seam is a real crate boundary.** `qe-hedger` and `qe-edge` do NOT depend on each other in production — firewall forbids it both ways; edge's `Cargo.toml` lists `qe-hedger` only under `[dev-dependencies]` (the `edge.rs` e2e test drives a real `HedgePlanner`), which the firewall (normal-deps only) correctly excludes. The order path compiles independently of the planner — two processes over the QE-218 wire.
5. **Facade / zero consumer churn.** `qe-runtime` re-exports via `pub use qe_edge::*; pub use qe_hedger::*;` (globs carry flat types + `pub mod` paths; contract types reach it through `qe_hedger`'s re-export of core); no collision (disjoint names, private `mod classify` in each). `qe-cli`/`qe-server` compile unchanged. Dropped `qe-storage` dep genuinely unused (grep).
6. **Full gate / no behaviour change.** Builds + tests + firewall + clippy green; the intra-doc-link fix (`crate::hedger::PositionKeeper` → `qe_runtime_core::PositionKeeper`) is move-required; Cargo.lock reflects only new crate wiring; no wire/golden change (restart_parity byte-parity corroborates).

### Non-blocking
1. The facade uses glob re-exports (`pub use qe_edge::*; pub use qe_hedger::*;`) rather than the prior enumerated list — correct today (disjoint names) but a future same-named public item in both crates would surface as a glob ambiguity at the facade. If the surface grows, consider re-enumerating flat type re-exports while keeping module globs.
2. `qe-runtime-core` ships with no tests of its own (a pure contract exercised through hedger/edge). Fine as-is; a doc-test on `TargetPosition::single`/`direction` would give the leaf crate independent coverage. Optional.
