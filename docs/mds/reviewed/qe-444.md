## Review verdict — [Approved] (0 blocking, 2 non-blocking)
1. **Directional (odd-in-side) — confirmed.** `Buy=+γ`, `Sell=−γ`; test asserts `Buy==−Sell` AND `Buy!=Sell` (not accidentally even). `realized_alpha_loss` signs adverse drift by direction. Applied cost `notional·γ` is the always-adverse magnitude of the odd drift (equivalent to a directional price shift — shortfall is never favourable), reducing net PnL for **both** a long and a short entry (both non-vacuous, same trade sequence).
2. **Default-0 serde-skip byte-identity — verified both directions.** At 0 the field is omitted (`!json.contains("alpha_loss")`) → calibration byte-identical to pre-QE-444 → `content_hash` unmoved, no `VINTAGE_FORMAT_VERSION` bump; old JSON round-trips (default→0→omit). `git diff` shows no fixture/golden; regenerate → empty diff; γ=0 backtest byte-identical.
3. **Composition preserved.** Kept separate from `cost`/`notional_cost` → QE-431 friction↔capacity parity untouched (`slippage_parity`); QE-435 `money_model_parity` passes; QE-432 oracle mirrors the term AND draws a non-zero coefficient (`0..0.005`) so it exercises the directional path; QE-440 composes.
4. **Determinism** (exact Decimal, no f64 on the sealed path); **measurement hook** sound & honestly deferred (mean clamped at 0 — favourable drift not credited); **firewall clean** (`alpha_loss` in qe-risk leaf, friction-only, no `qe-wfo→qe-ensemble` edge).
