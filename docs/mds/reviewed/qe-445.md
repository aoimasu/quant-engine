## Review verdict — [Approved] (0 blocking, 2 non-blocking)
1. **Genuine parallelism confirmed.** Bounded cooperative wait (spin ≤200k until `max_live≥2`, else timeout) → 8 threads reliably overlap (`conc_many≥2`), 1 thread times out with no deadlock (`conc_one==1`). Serial pool would fail `conc_many≥2`, so byte-identity is earned under real overlap.
2. **Assertion load-bearing (mutation guard).** `per_task_artifact` (`task_rng(master, gi)` indexed by genome) is execution-order-invariant (`forward==reversed`); `shared_rng_artifact` (one shared RNG in execution order) is order-sensitive (`forward!=reversed`). The AC test runs the real production `evaluate_and_insert` (verified: `.par_iter().enumerate().map(|(index,g)| task_rng(master_seed, index))`) at 1 vs 8 threads → byte-identical, and the guard proves a shared RNG would break it.
3. **Correct stage; complete artifact.** `serialize_archive` compares both directions in fixed order, cells in sorted BTreeMap order, canonical genome JSON + fitness bit pattern per elite.
4. **Test-only; no golden.** `git diff --name-only` = test + note + `Cargo.toml` dev-deps + `Cargo.lock`; regenerate → empty diff.
5. **Dev-dep cycle clean.** `qe-determinism --dev--> qe-wfo --> qe-determinism`: new deps under `[dev-dependencies]` (firewall parser excludes them → no forbidden production edge); `cargo metadata` resolves cleanly (cargo permits dev-dep cycles); 3 new tests RUN (not ignored) and pass.

### Non-blocking follow-ups (accepted)
1. `conc_many >= 2` is a bounded (non-hanging) **timing** assertion — highly reliable at 8 threads/160 tasks and times out gracefully, but on a pathologically loaded/single-core host the overlap could rarely not materialise within the 200k-spin budget → a possible false-negative. **Relevant when CI is re-enabled** (a loaded runner) — worth widening the budget or gating the concurrency assertion on `available_parallelism() > 1`.
2. The mutation guard uses local reimplementations mirroring the two seeding schemes (the correct technique; the AC test covers production).
