## Decision summary

New crate `qe-report` (most-downstream consumer; deps only `qe-validation` + `serde`). `VintageReport`
embeds the real `qe_validation::RobustnessReport` (DSR/PBO/SPA) and represents the other sections
(performance, cost-sensitivity, per-regime, correlation, capacity, worst-case loss) as plain serde rows.
`render_markdown()` is a pure function (byte-identical); the report round-trips through serde.
`pairwise_correlation_summary` computes min/median/max/mean of all C(n,2) pairwise Pearson correlations.

## Review (pass 1) — Approved

Principal-level pass. Verdict: **Approved.** No real bugs.

- **(a) Completeness** — `render_markdown` emits all seven backlog sections in fixed order; the
  all-sections test asserts each by needle (incl. the `| 2.00 |` 2× row and `binding scenario: gap`).
- **(b) Reproducibility** — pure function of `self` (fixed order, locale-independent `{:.N}`, iteration
  only over the report's own `Vec`s, no clock/RNG/HashMap); two renders byte-identical; serde round-trip
  holds (embedded `RobustnessReport` is serde).
- **(c) `pairwise_correlation_summary`** — textbook Pearson (NaN-free); fixture hand-verified
  (`[+1,−1,−1]` → min −1, max +1, mean −1/3, median −1); `total_cmp` sort sound; `<2`-series zeroed.
- **(d) Firewall** — deps only `qe-validation` + `serde` (no `qe-wfo`/`qe-ensemble`); not an upstream in
  `firewall_rules()`; QE-132 guard still passes with the crate added.
- **(e) Design** — embedding the real `RobustnessReport` while caller-populating the other sections as
  plain rows is the right dep-light choice for a leaf presentation crate.

Two non-blocking notes: (1) `cost_sensitivity`/`regime_expectancy` are caller-populated `Vec`s not
structurally forced to contain the 1×/2× rows — **addressed** in `84b46dc` with a caller-contract doc on
`cost_sensitivity`. (2) a non-finite `f64` would make `serde_json` error — harmless, not the report's job
to validate (no action).
