### Goal
*(Reviewer-added.)* Before any capital, run the full loop against live data computing **would-be** orders with
**no submission**, reconciled vs the simulator — catching wss-stitch, mark-EMA, netting, cutover bugs.

### Acceptance criteria (from backlog)
- [x] A shadow run over a defined live period produces would-be orders that **reconcile with the simulator
  within tolerance**; **no orders are submitted** — `shadow_run_reconciles_with_simulator_and_submits_nothing`.

### Implementation summary
- `crates/runtime/src/shadow.rs`. **`ShadowGateway`** = the Edge gateway in dry-run: `observe(&rev)` runs the
  same `plan_delta` the live edge does but **logs** the `WouldBeOrder` and advances a shadow position
  as-if-filled, submitting nothing (`orders_submitted()` is a literal `0` — no submit path exists).
  **`ShadowRun`** drives each `TargetRevision` through both the shadow edge and a submitting reference QE-218
  `PlannerAdapterLink`, reconciling the two positions via the QE-221 `ReconciliationGuard` (`AlarmOnly` — a
  dry-run reports, never halts); `report()` yields `ShadowReport`.
- `ShadowRun::observe_marks(shadow_mark, reference_mark)` feeds the shadow edge (live pipeline under test) and
  the reference keeper (venue truth) marks independently, so a mark-pipeline drift produces a real divergence
  **through the gate's own API**; `observe_mark(m)` is the aligned shorthand. `reconciled` is a run-level latch.

### Review notes — [Approved] (two passes)

**Pass 1 (@ `d2ea50b8`) — [Reviewed].** Verified correct: "no submission" is structural (`ShadowGateway` has no
submit path; `orders_submitted()` const `0`, asserted); `AlarmOnly` right for a dry-run; const-`0` beats a
mutable flag; `would_be_orders_match_simulator_fills` + `at_target_revision_logs_no_would_be_order`
non-vacuous; determinism/encapsulation/no-new-dep/firewall clean; synthetic feed a fair layer boundary.
**F1 (blocker):** inside `ShadowRun` the shadow and reference are fed identical inputs and the sim does full
fills, so `report.reconciled` was **constant-true** — the gate's red state was unreachable through its own API
and the AC reconcile assertion vacuous; the old divergence test hand-wired separate objects (proved the guard,
not the gate). For a capital-blocking gate the fail path must be reachable and tested through the gate.
**F2 (note):** the oracle shares `plan_delta` with the code under test — a bug inside `plan_delta` corrupts
both sides identically; document as a known blind spot.

**Pass 2 (@ `f76045f2`) — [Approved].** F1 resolved: `observe_marks(shadow_mark, reference_mark)` feeds the two
sides independently, so a mark-pipeline drift yields a real divergence **through `ShadowRun`**;
`gate_reports_a_mark_pipeline_divergence_through_shadow_run` drives it and asserts `reconciled == false`,
`max_divergence == 0.05 > tolerance`, shadow `0.25` vs sim `0.20`, `orders_submitted == 0` on the fail path.
`a_divergence_latches_the_run_red` adds correct run-level latch semantics (`reconciled` only ever set false; a
re-converging step keeps the run red with peak `max_divergence`). The happy-path AC test is now a *meaningful*
green because the red state is reachable. F2 surfaced as a deliberate scope boundary in the design Risks. "No
submission" stays structural; determinism/encapsulation/no-new-dep/firewall preserved. The gate genuinely
bites. Approved — the final P2 ticket / the gate blocking Phase 3 live capital.
