## Review verdict — [Approved]
Fix proven non-vacuously (un-seeded control stays silent; seeded anchor == `CommittedPeak::from_series` = 200; first tick at 170 trips Med). Reset/rollover rule provably monotone: `observe` keeps `peak = Some(p).max(equity)` (running max), `reset` sets `seed_peak = self.peak` without lowering `peak`, so the anchor is carried forward at `max(seed, highest observed)` and can never be reduced — a restart/rollover cannot silently re-arm a near-tripping breaker. Un-seeded breakers keep legacy `peak=None`; `BreakerLayer::reset` still applies the QE-416 fail-safe pre-gate. `seed_committed_peaks` is panic-free on the deny-order path (`if let (Some,Some)` + `get_mut`, None skipped, out-of-range ignored). Behavior-preserving default (`CircuitBreaker::new` stays `seed_peak:None`; `restart_parity` + existing tests unchanged). Deterministic (pure fn of committed peaks); firewall green.
