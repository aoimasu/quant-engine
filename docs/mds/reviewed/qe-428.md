## Review verdict — [Approved]
1. **Reporting matches selection (core).** `backtest_config` no longer pins `impact: Decimal::ZERO`; it defaults slippage to `SlippageModel::default()` (impact `0.0001`), and `reporting_impact = Some(v)` overrides only the impact coefficient (half-spread stays default). `reporting_impact_defaults_to_selection_impact` proves the match literally: reporting default equals `qe_wfo::backtest::BacktestConfig::default().friction.slippage.impact` AND `!= ZERO`. Selection genuinely prices this flat friction coefficient; the QE-128 capacity model weights allocations, not the impact coefficient, so leaving it out of impact is correct, not a miss.
2. **Golden regeneration integrity (critical) — proven by reproduction.** `regenerate_fixtures` (the `#[ignore]`d test) builds the golden from real `run_backtest(...)` output via `serde_json::to_string_pretty` (not hand-constructed). Reviewer re-ran it: regenerated SHA-256 `af182509bef38e66d4dcb5351a4d385fb7e1b741be6aed6b6077052629ed0d65` — byte-identical to the committed golden — so it is deterministic/reproducible and genuinely machine-generated. The field diff moves ONLY cost-impacted fields (`metrics.{cagr,sharpe,sortino,max_dd}`, `equity_curve`, `drawdown`, `monthly_returns`); structurally-invariant fields (all 16 trade rows, `costs`, `window`, `universe`, `win_rate`, `profit_factor`) are byte-identical. `sample_vintage.json` + train/vintage goldens untouched.
3. **Direction sanity.** Priced impact reduces reported net performance as expected: `cagr` 0.96347→0.96226 (↓), `sharpe` 1.53096→1.52980 (↓), `sortino` 2.18216→2.18049 (↓), `max_dd` −0.042994→−0.043002 (worse). ~0.13% relative on CAGR — plausible for a 1bp impact on this fixture; no sign flip, no implausible jump. `reporting_prices_impact_lowers_returns_vs_zero_impact` corroborates (default-priced vs `Some(ZERO)` over the same store → identical trade count, strictly lower final equity + CAGR).
4. **CLI flag correctness.** `--reporting-impact` None→selection default, `Some(v)` override, `Some(0)` reproduces the legacy golden (clean rollback/escape hatch); panic-free via `Decimal::from_str_exact` → `CliError::Usage`; parse tests cover absent + `0.0002`→Some.
5. **Scope.** Confined to `crates/cli` (backtest job + args + wiring) + tests + regenerated golden + design note (`train_job` +3 is just the `reporting_impact:None` field); selection untouched; no Cargo/deps/firewall change (`dependency_topology` passes).
