## Decision summary

Three competing emitters — `LocalRefine` (exploit), `Explore` / `FreshRandom` (explore) — selected by a
sliding-window reward bandit (`OperatorSelector`): credit = mean of an operator's last `WINDOW` rewards;
selection probability ∝ `max(credit,0) + EPSILON` (exploration floor → no starvation), deterministic via
a seeded `DetRng` (QE-006). Exploration-when-sparse / exploitation-when-dense is **emergent** from the
reward (D3), proven by a sparse vs dense simulation using identical machinery. Credit signal is
**in-training only** (D4): `ApplicationOutcome` (`NewCell` / `ImprovedElite{gain}` / `NoImprovement`)
has no OOS field by construction; QE-121/QE-132 enforce the same downstream.
