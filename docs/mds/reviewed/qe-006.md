## Acceptance criteria (met)

## Implementation
New crate `qe-determinism` — `rng` (portable ChaCha8 `DetRng` + per-task seed derivation `task_rng`
via SplitMix64, so a task's stream depends only on its index, never on scheduling), `harness`
(`reproduce`/`is_reproducible` — re-run a stage twice, assert byte-identical), `lineage`
(`Lineage{config_hash,input_snapshot_id,code_commit,seeds}` with stable SHA-256 `id()`, `from_config`
folding QE-002's `content_hash`, `Artifact<T>`/`HasLineage`).

## Review outcome
**Approved** (round 2). Round 1 approved with an adversarial discrimination check: the reviewer wrote
a shared-RNG anti-pattern probe (`Mutex<DetRng>` inside `into_par_iter`) and confirmed it **diverges**
across thread counts, proving the AC#1 tests are not vacuous — they pass precisely because `task_rng`
removes the schedule dependence.

Two non-blocking advisories were raised; in round 2:
1. **Golden value (resolved).** Added platform-independent golden assertions —
   `rng::tests::golden_stream_and_derivation_are_pinned` pins `seed_rng(0).next_u64()`,
   `derive_seed(0,0)`, `task_rng(0,0).next_u64()`; the integration test asserts a hard SHA-256
   (`94d840f3…a038`) of the 4096-task draw. A `rand_chacha` bump or any SplitMix64-derivation change
   now fails loudly instead of silently re-baselining vintages.
2. **Shared `hex` util (deferred).** `hex(&[u8])` duplicates the one in `qe-config`; folding into a
   shared util crate would rewire already-merged `qe-config` — out of scope for a determinism ticket,
   tracked as a small follow-up.

Reviewer re-confirmed **[Approved]** after the golden additions (verified genuine: the pinned values
are exactly what the implementation emits, since the tests pass).
