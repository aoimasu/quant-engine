### Goal
Convert the codebase's hand-maintained "no bare production `unwrap`" convention into a compiler-enforced invariant via the clippy `restriction` lint `unwrap_used`, which is not part of the default `all` group.

### Acceptance criteria (from `docs/backlog.md` QE-267)
- [x] `cargo clippy --workspace --all-targets` stays green with the new `deny`.
- [x] A freshly-introduced **production** `.unwrap()` fails the gate.
- [x] Test-module unwraps are unaffected.

### Implementation summary
- Root `Cargo.toml`: added `unwrap_used = "deny"` to `[workspace.lints.clippy]` (`expect_used` deliberately left off — that is QE-268).
- New `clippy.toml` at workspace root: `allow-unwrap-in-tests = true`.
- `crates/cli/src/jobs/metrics.rs`: lone bare `*equity.last().unwrap()` → `.expect("equity.len() >= 2 verified above")`.
- Backlog: QE-267 section inserted after QE-266.

### Deviation evaluated & accepted
`allow-unwrap-in-tests = true` only exempts code inside `#[test]` fns / `#[cfg(test)]` modules — NOT module-level helper fns in separate `crates/*/tests/*.rs` integration files, which the mandated `--all-targets` gate compiles. Those failed under the new `deny`. Resolution applied: a per-file `#![allow(clippy::unwrap_used)]` added to the **17** integration-test files that use `.unwrap()` (test-only code; no production behaviour change). Production `src/` remains fully governed by the workspace `deny`. Diff: 22 files, +205/−1.
