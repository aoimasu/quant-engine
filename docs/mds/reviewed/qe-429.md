## Review verdict — [Approved]
Trading-critical latent-wiring closeout, reviewed adversarially.

1. **Live wiring is REAL at the cutover constructor.** `Cutover::from_reconstructed_calibrated` builds `from_calibration(calibration, strategy_ids(), fast_window)` + `seed_committed_peaks(state)` BEFORE any tick, and `feed_live_bar` routes `clamp` before netting — so the previously test-only breaker pieces now have a real assembler at the exact calibration↔reconstructed-state convergence point. **Honest key finding:** the new constructor is invoked only by the integration test — but so are the legacy `Cutover::new`/`from_reconstructed` (grep: the only non-test refs are the classify error-type + lib re-export), because **no live binary runs any cutover yet** (the two-process live entry is future work, out of scope). So QE-429 does not relocate latency to a worse state; it moves the wiring from "pieces have no assembler" to "real cutover constructor assembles + seeds + routes them, exercised at the actual call site" — as far down as it can go, transparently documented (design note §1).
2. **Seed / keying / fail-safe (core).** Keyed by the `strategy_ids()` SSOT delegate (no calibrated member pre-gated); the AC test seeds peak 200 → first tick at 170 trips Med, an uncalibrated strat is fail-safe pre-gated, and `feed_live_bar` clamps both gated legs to Exit BEFORE netting. The unseeded control (empty `ReconstructedState`) stays silent — the QE-401 bug — proving the seed is load-bearing (non-vacuous).
3. **Behaviour preserved.** Legacy path `breaker: None` → `feed_live_bar` skips clamp (true no-op, proven); all 82 hedger tests unchanged.
4. **Vintage-hash safe.** `MarkEmaConfig::Default` reproduces the spec baseline tick-for-tick (proven over a gappy series), runtime-only, never serialized. The QE-416 seal constants are genuinely untouched (only doc lines in the diff); `sealed_vintage_…calibrates_every_strategy` passes → `content_hash` unmoved. The deferral is CORRECT (their outputs feed `content_hash` and their config home `qe_config::Config::content_hash` feeds the vintage lineage, so promoting would move the hash), not a cop-out.
5. **Panic-free.** `cutover.rs` gains the QE-268 deny lint; uses `.ok_or()?` not unwrap; `debug_assert! ≠ panic!`; clippy clean.
6. **Firewall + scope.** Firewall 1/1, no new edge, confined to `crates/hedger` + the integration test in `crates/runtime/tests/`, no seal/lineage/schema change.
