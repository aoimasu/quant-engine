### Acceptance criteria (from backlog)

### Implementation summary
- `qe-cli train` rewritten (`crates/cli/src/jobs/train.rs`, mirroring `backtest.rs`) to run the REAL pipeline:
  seeded MAP-Elites/WFO search (sequential `VariationDriver::step` over one shared `MapElitesArchive`) →
  `elite_pool` → `qe_ensemble::search_portfolio` (discrete-DE) → `qe_validation::assess` (DSR/PBO/SPA) →
  `qe_gate::evaluate_g1` → `qe_vintage::Vintage::seal` + `VintageRepository::write`. The QE-013 manifest-only
  path removed from `lib.rs`. Single-threaded; seeded via `qe-determinism`; configurable small budget.
- **Rich JSON-line progress** (serde-tagged `"t"`, additive): `gen` (generation/coverage via
  `qe_wfo::regularise::coverage`/coverage_long/short/best_fitness), `ensemble` (folds/members/score),
  `gate` (promoted/failed[]/in_sample_sharpe/holdout_sharpe/dsr/spa_pvalue/n_trials from the real G1Decision),
  terminal `done`(+`vintage` id, `Option` `skip_serializing_if` so backtest's `done` byte-shape is unchanged)
  or `error`. `TrainResultDoc` sidecar carries G1Decision + RobustnessReport + budget.
- **Catalogue-schema aligned with QE-251:** evolves against the same `features.rs::catalogue_schema()` =
  `FeatureSchema::from_catalogue(&CatalogueConfig::default())` `{states:5}`.

### Review — [Approved] (2 passes; 0 blocking)
- Pipeline REAL not stubbed (traced search→ensemble→validation→G1→seal). Determinism genuine: `vintage_id` =
  lineage (seed-folded, deterministic by construction) + `content_hash` pins the seeded single-threaded search;
  the one rayon site (`mapelites.rs:281 evaluate_batch`) is off this path AND thread-count-invariant. No
  wall-clock/unseeded RNG. `train_is_deterministic_for_a_fixed_seed` asserts both hashes.
- Catalogue alignment is a real cross-check (`sealed_vintage_is_backtestable_by_qe251` seals then runs QE-251
  `run_backtest`, asserting `Ok` + non-empty equity). G1 semantics correct: gate fields from real G1Decision
  (nothing hardcoded); vintage sealed regardless of verdict but `VintageContent` has no `promoted` bit — the
  verdict lives in the gate progress line + result.json sidecar, so a failing gate can't masquerade as promotable.
- Pass-2: nit a fixed — `DEFAULT_TRAIN_HOLDOUT` 30→31 (30 holdout returns ≥ G1 `min_holdout_samples=30`), so
  `holdout_has_sufficient_samples` is satisfiable at the default budget.
