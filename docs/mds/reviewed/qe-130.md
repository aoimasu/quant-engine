## Review (pass 1) ‚Äî Approved

Principal-level pass. Verdict: **Approved.** No real bugs.

- **(a) AC.** `worst_case_loss: Option<f64>` is a field of the hashed `VintageContent` ‚Äî round-trips and
  is in the hash (`0.40` vs `0.28` ‚Üí different hash; negative rejected at seal). `VINTAGE_FORMAT_VERSION`
  1‚Üí2 correct per the QE-129 contract. Firewall preserved ‚Äî grep-confirmed **no `qe-ensemble`** in
  `crates/vintage/Cargo.toml`. Bookkeeping commit `58b4f0f` docs-only.
- **(b) Math hand-checked.** `max_drawdown` reports the single worst trough and holds it through a
  post-trough recovery (`[0.1,‚àí0.3,0.05]`‚Üí0.3). `weighted_combined` uses actual weights, truncates to
  shortest. Compound-at-trough reproduces all four test values exactly (gap 0.28, funding 0.232, ADL
  0.24, window 0.5). The conservative shock-lands-while-down model is right for a pre-declared max-loss.
- **(c) Aggregation.** True `max_by(total_cmp)`; binding scenario is the real argmax; breakdown complete;
  empty set ‚Üí `(0.0, "")`. `HistoricalWindow` slicing panic-proof (`saturating_add` + `.min(len)` both
  bounds, `lo ‚â§ hi`).
- **(d/e) Monotone; no negative-loss path (`.max(0.0)` clamps); negative weights counted via `abs`;
  NaN-robust (`f64::max` ignores NaN); pure/deterministic, no RNG.**

One non-blocking note, no action: a combined return `< ‚àí1` (only reachable with leveraged `Œ£|w|>1`, which
