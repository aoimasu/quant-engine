### Goal
*(Reviewer-added.)* Reconciliation should not be post-hoc only; a live position mismatch beyond tolerance
should be a fast safety check that can trip the kill-switch.

### Acceptance criteria (from backlog)
- [x] An injected position desync beyond tolerance raises an alarm and can halt —
  `sustained_divergence_alarms_and_halts_after_threshold`.

### Implementation summary
- `crates/runtime/src/reconciliation.rs`: `ReconciliationGuard::check(expected, &PositionReport)` compares the
  runtime's expected signed position against the venue's authoritative report each period. `delta = |expected −
  venue|`; within tolerance → `Reconciled` (streak resets); beyond → an alarm (`alarms += 1`, streak `+= 1`)
  and, under `AlarmAction::HaltAfter { consecutive }`, trips the QE-216 `KillHandle` once the divergence has
  **persisted** for `consecutive` consecutive checks (debounce). `AlarmOnly` never halts;
  `halt_immediately()` = `HaltAfter { 1 }` (quiescent-point single-check halt).
- Sign-aware (a flip sums magnitudes), latching (QE-009 kill), fails safe (negative tolerance clamped to 0,
  `consecutive.max(1)`). Detector only — attribution is QE-302; takes `expected` explicitly (no circular
  self-check).

### Review notes — [Approved] (two passes)

**Pass 1 (@ `b4789c58`) — [Reviewed].** Verified correct: divergence math (`signed_qty` Long/Short/None, `delta
= |expected − venue|`, inclusive tolerance, sign-flip summing magnitudes, phantom-position detection, all in
`Decimal`), fail-safe clamp, out-of-band kill trip via held clone, latching, non-vacuous tests. **F1
(blocker):** a stateless single-check auto-halt would false-halt on routine in-flight-order skew — a
timer-driven check sees `expected` ahead of the venue's not-yet-reported fill, so `delta` briefly equals the
in-flight quantity and auto-halts the book on a benign propagation blip; neither a wider tolerance (blinds the
detector) nor `AlarmOnly` (abandons the AC) fixes it. Required a consecutive-breach (debounce) threshold.
**F2 (nit):** absolute-contracts tolerance is coarse across scales (deferred), and must not be widened to mask
the transient.

**Pass 2 (@ `f2693ea5`) — [Approved].** F1 resolved: `AlarmAction::Halt` → `HaltAfter { consecutive }`;
`check_qty` resets `streak` on any reconciled check and trips only when `streak >= consecutive.max(1)`. A
transient breach→reconcile→breach never reaches the threshold (`transient_single_period_skew_does_not_halt`),
while two consecutive breaches halt (`sustained_divergence_alarms_and_halts_after_threshold`) — fail-safe
preserved. Fail-safe edges intact (`HaltAfter{0}` halts after one via `max(1)`; negative tolerance clamps;
kill latches independently of the streak so a post-halt reconcile cannot un-halt). All prior correctness
preserved; 10 non-vacuous tests. F2 correctly deferred and solved via the streak lever, not a widened
tolerance. Approved.
