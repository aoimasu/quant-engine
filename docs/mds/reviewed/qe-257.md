### Implementation summary
- **Coverage relocation (discharged the QE-253 carried requirement):** `coverage()`/`CoverageRow` moved
  `qe-cli::jobs::ingest` → `crates/storage/src/coverage.rs` (error `RunError`→`StorageError`, behavior-identical);
  new `MarketStore::bar_instruments()` key-only deterministic enumeration (contiguous lex keys + running-last
  dedupe) backs `coverage_all`. `qe-cli` keeps `pub use qe_storage::{coverage, CoverageRow};` so QE-253 tests
  are untouched and pass.
- `GET /api/vintages` — `VintageRepository::list()` (sealed/hash-verified only, skips strays, missing dir ⇒ [],
  ascending) → `{id,label,summary{chromosomes,content_hash,worst_case_loss,format_version}}` (shape for QE-259).
- `GET /api/market-data/coverage` — relocated `qe_storage::coverage()` over the configured market store.
- Both `.merge`d into `auth::protected_routes()` before `route_layer(require_session)` — session-gated (401 without).
- `ServerConfig` extended: `artifacts_dir`, `market_dir` (`QE_SERVER_*`, relative defaults); `ReadState` + `FromRef`;
  handlers use `spawn_blocking`; store opened once at startup.
- Fixtures copied into `crates/server/tests/fixtures/` (server can't dep qe-cli); store copied to tempdir before open.
- **deny.toml: NONE**; firewall green (full transitive trace: `qe-server → {qe-storage,qe-vintage}` closure has
  no `qe-runtime`/`qe-venue` edge; both QE-254 guards untouched + pass).
