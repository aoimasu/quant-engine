## Review verdict — [Approved] (impl 52e5d30) + [Approved] (reframe 7b27d67)
`qe_wfo::cv_fitness::fold_isolation_fitness` backtests each genome on disjoint PurgedKFold test folds in isolation (flat start) and reduces per-fold net returns to `mean ± SE` (fitness = mean), replacing the old contiguous position-bleeding adjacent-slice windows — a genuine leakage improvement satisfying the spec's "wire noise-robust windows to CV folds not adjacent slices." Given the fixed-rule architecture (no per-fold refit) and the ~87-bar/lookback-34 fixture budget, textbook train-purged OOS is semantically inapplicable and infeasible; the G1 terminal holdout remains the only true OOS gate (out of scope). Both ACs non-vacuous. Config `SelectionConfig.cv_folds` default 4 (fixtures 2), Σ|test_k|=n so per-genome cost ~unchanged. Determinism preserved (fixed fold boundaries/order, no RNG). PurgedKFold over WalkForward appropriate at this budget.
