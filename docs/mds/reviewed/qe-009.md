## Implementation
New crate `qe-risk` (contract, not enforcement — QE-215/216): `LimitKind` + per-kind `LimitOutcome`
policy (`default_outcome`), validated `Leverage`/`Fraction` caps (rejected on construction AND serde),
`RiskLimits`, out-of-band latching `KillHandle`/`KillSwitch`, and the `OrderGate` contract. The kill
check is **structural**: implementors write `admit_within_limits`, and the provided `admit`
kill-prechecks first. `assert_honours_kill_switch` is the reusable conformance check. `qe-runtime`
references the contract via `OrderPort: qe_risk::OrderGate` with a runtime conformance test.

## Review outcome
**Approved** (round 2).

- **Round 1 — [Reviewed], one blocker.** Verified positively: genuine AC #1 supertrait reference;
  concurrency-sound latching `SeqCst` `KillHandle` (reason written under the mutex before the tripped
  store; not serde-deserializable); validated `Leverage`/`Fraction` serde (QE-007 lesson held);
  sensible `default_outcome` policy. **Blocker:** `assert_honours_kill_switch` exercised only the
  provided helpers, never `admit` — a `BadGate` whose `admit` ignored the kill passed conformance
  (convention-vs-enforcement trap).
- **Round 2 — [Approved].** Fixed both ways: `admit` is now a **provided default** that
  kill-prechecks then delegates to a required `admit_within_limits` (structural guarantee); the
  conformance check now also asserts `gate.admit(&intent)` is `FlattenAndHalt` once tripped. Reviewer
  re-ran the round-1 attack — the `BadGate` override now **fails** conformance; a gate implementing
  only `admit_within_limits` honours the kill for free. Hole closed.
