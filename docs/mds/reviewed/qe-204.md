### Goal
Fills, position reports, and heartbeat are the authoritative ground-truth feed for the Position keeper.
Subscribe the subaccount-scoped private user-data stream; reconnect with listen-key renewal; simulator
equivalent for sim mode.

### Acceptance criteria (from backlog)
- [x] Fills/positions/heartbeat delivered **in order** тАФ `events_are_delivered_in_order`.
- [x] A dropped stream reconnects **without losing position truth** (re-snapshot on reconnect) тАФ
  `reconnect_renews_key_and_resnapshots_without_losing_position_truth` (+ listen-key-expiry variant).

### Implementation summary
- `userdata` module: event model (Fill / PositionReport / PositionSnapshot / Heartbeat /
  ListenKeyExpired / Snapshot) over three seams тАФ `ListenKeyProvider`, `UserDataConnector` +
  pull-based `UserDataConnection::poll`, `PositionSnapshotSource`.
- `UserDataSession`: `connect()` establishes an initial snapshot; `pump()` delivers events in order and,
  on disconnect **or** listen-key expiry, renews key + reconnects + re-snapshots (`reconnected = true`,
  fresh `Snapshot` event). `keepalive()` holds the current key.
- `sim::ScriptedUserData`: concrete sim-mode implementation of all three seams (not test-only).
- Real signed-REST listen-key client + async wss adapter deferred to runtime wiring (per QE-202).
