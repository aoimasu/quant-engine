Meets all AC; no blocking issues. Verified against QE-465.md AC, design §8.3/§8.2, the QE-464
`create_ingest`/`IngestParams`/`CoverageRow` server contract, the diff, and a full web test run (26 files / 107 tests green).

1. **POST body matches `create_ingest` EXACTLY — CONFIRMED (highest-risk).** Server `create_ingest` takes `Json<serde_json::Value>` — the BARE `IngestParams`, wrapped to `type:"ingest"` internally. Client `createIngestRun` posts bare `IngestParams` to `/api/ingest` via `postCreateRun` (NOT the `{type,params}` envelope); field names match one-for-one. `NewIngest.test` asserts `body.type` is `undefined` + every field → a production silent-400 is ruled out.
2. **Provenance column honesty — CONFIRMED (headline).** `ProvenanceCell` badges every row (`real`→up, `synthetic`→loud amber, `unknown`→neutral, fail-safe `?? unknown`, `unknown` never softened to `real`). Client does NO merging. Interleaved test seals real→synthetic→real, asserts 3 rows / 2 REAL / 1 SYNTHETIC, and iterates EVERY body row asserting exactly one `/^(REAL|SYNTHETIC|UNKNOWN)$/` badge.
3. **Cancel actually cancels — CONFIRMED.** `doHalt` → `haltRun(runId)` → run-type-agnostic `POST /api/runs/{id}/halt`; test asserts the POST fires.
4. **Progress honesty — CONFIRMED.** Standard coarse `<RunProgress>` from real `meta.progress`; no fabricated %. Per-page progress flagged as a follow-up (needs the HistoricalSource streaming seam).
5. **Client hints only; server authoritative — CONFIRMED.** `validate` mirrors `validate_ingest`'s either/or but only removes the affordance; server `400` surfaced inline. Web-only diff; Rust gate trivially green.
6. **`synthetic:false`-always — ACCEPTABLE (positive design).** Operator trigger is the real-ingest path; synthetic stays the CLI generator — the more honest choice (no SPA-populated store is ambiguously synthetic).
