## Decision summary

`search_portfolio` is DE/rand/1/bin over the QE-115 `EnsembleMask` with greedy (elitist) selection — the
best-so-far is monotonic (the "convergence"). `cross_val_score` partitions the common time axis into
`folds`, takes `leave_one_out_min` (the QE-115/D6 wide-basin floor) per fold, and returns the minimum
across folds (worst-fold, worst-member-dropped) — a robust basin. Scoring is on the pool's net-of-cost
return series. One seeded `DetRng` → byte-deterministic. `qe-determinism` promoted to a normal dep.

## Review (pass 1) — Approved

Principal-level pass. Verdict: **Approved.** No real bugs.

**Firewall (QE-001/QE-132) intact.** `qe-ensemble` deps add only `qe-determinism` — no `qe-wfo`; the only
`qe-wfo` strings are doc-comments asserting its absence.

- **(a) Convergence genuine, not gamed.** Hand-traced the fixture: three diversifiers each spike `+0.02`
  on two of three period-slots; the triple is constant `0.0133`/bar; pairwise correlations are negative so
  the corr penalty floors at 0. The `leave_one_out_min` floor makes a *pair* drop to a singleton (worst-slot
  CVaR ≈ 0) → CV score ties the best singleton; only the *triple*'s LOO (drop to a pair, CVaR `0.01`) clears
  it. So `count ≥ 2` AND `score > best_singleton` only pass *together* if the DE discovers the full 3-member
  basin; the bad strategy (index 3, negative mean + decile `−0.4` shock) is excluded; monotone `history` is
  a real consequence of greedy `≥` replacement. A no-op/broken/non-elitist search fails one of the asserts.
- **(b) Net-of-cost.** Pool series are the net returns the objective consumes directly; the
  cost-subtraction test (`net < gross`) confirms the score is a function of those returns.
- **(c) `cross_val_score`.** Correct min-across-folds of `leave_one_out_min`; `k = folds.max(1).min(t)`
  makes the `hi<=lo` / `hi.min(len)` guards dead-defensive (never mis-slice); `worst` never returned as the
  `INFINITY` sentinel.
- **(d) DE loop.** `np = pop_size.max(4)` removes any `pick_three_distinct` infinite-loop risk;
  `repair_nonempty` keeps trials non-empty; determinism via a single `seed_rng(seed)` stepped sequentially.
- **(e) Edges.** Empty pool, single strategy, `folds > t` handled.

Two non-blocking observations (dead-defensive guards; `repair_nonempty`'s content-coupled rng draw — NOT a
determinism bug, confirmed by the same-seed full-struct equality test). No blocking issues.

No outstanding comments. Approved & merged.
