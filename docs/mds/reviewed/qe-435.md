## Implementation summary — and the finding
Oracle parity test. **Finding (reviewer-confirmed TRUE):** the two models agree exactly on the **fill** (`notional/mark` ≡ `target/mark − current`), but the live side has **no cost model** — `VenueSimulator::submit` fills at the raw mark with zero fee/slippage and holds no cash/PnL state; `EvaluatorSession` emits decisions only. Cost on the selection path lives in exactly one place — `qe-wfo` friction — which (QE-431) derives from the single content-addressed `SlippageCalibration`, so there is **no divergent second live number**. The test (1) proves fill-geometry parity over 7 cases through both real code paths, (2) proves wfo cost reduces exactly to the shared calibration at `reference_mark`, (3) quantifies + asserts the one real gap (sim charges 0; wfo charges `taker_fee + calibration_slippage`). Adding a cost model to `VenueSimulator` deliberately deferred (QE-268 panic-free order path, golden-moving, no correctness gain until the live path prices money).
