**Fix pass (`b4c1e12`) — legacy-provenance fail-safe applied (this reviewer's top recommendation, required before merge).**
`store_data_provenance` now maps `Unknown`/`Empty` → `DataProvenance::Mixed` ("not verified pure-real") instead of `Real`:
in this deployment all pre-QE-464 bars came from `--synthetic`, so legacy `unknown` stores would otherwise stamp a
confident `real` vintage — the exact hazard QE-464 exists to kill. The extended test locks BOTH directions
(legacy/untagged → `Mixed`, pure-real → `Real`, no over-rotation). Scope clean (train.rs mapping + test; store.rs fmt-only),
no version bumps, coverage still key-only, gate green.
