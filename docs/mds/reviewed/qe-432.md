## Implementation
Independent, deliberately-naive reference oracle (test-only, zero production/dep change): (a) roll-up recomputed by a fresh O(n)-per-window scan, property-tested `== reconstruct_batch`; (b) net-of-cost cost-ledger + geometric fitness recomputed by a trade-by-trade exact-`Decimal` replay, property-tested `≡ backtest` (Decimal money byte-exact; f64 returns/fitness within `1e-9`). Mutation guards on both paths. **1024 seeded cases** (256×2 equivalence + 256×2 mutation) via `qe-determinism`.

## Review verdict — [Approved] (0 blocking, 3 non-blocking)
1. **Independence (CRITICAL) — CONFIRMED.** `reference_rollup` recomputes each coarse bar by its own fresh O(n) scan (own aggregation), never calling `reconstruct_batch`/the fold. `reference_backtest` reimplements the entire exact-`Decimal` cash/pos ledger, funding, per-bar returns, `net_pnl`, and windowed `mean ln(1+r)` from scratch, never calling `apply_fill`/`split_windows`/`NoiseRobustFitness`/`log_growth`. The only shared call is `Genome::decide` — correctly scoped out (signal decision, not the cost path under test). Optimised entry points are the real production `reconstruct_batch`/`backtest`, not doubles.
2. **Mutation guards — real & non-vacuous.** Production can't be line-edited from an external test, so each guard injects the bug into the reference's byte-faithful copy and asserts `reference == optimised` (faithful) AND `reference != mutant` (detectable) — transitively sound (production-with-bug would equal the mutant → the equivalence assertion would fail). Guards: volume `max`-vs-`Σ` fold typo; dropped slippage `cost_multiplier` (the per-trade cost bias DSR can't remove, forced with multiplier ≥ 2 so it bites). `caught` assertions confirmed to fire.
3. **Coverage & determinism — confirmed.** 1024 seeded cases from a fixed master seed; run twice → identical. Decimal money + counts byte-exact; per-bar `returns` compared on every bar (~30k points); only order-sensitive f64 uses the justified `1e-9` tolerance. Meaningful randomised space (targets M15..D1, gaps, random genomes/prices/friction/windows).
4. **Test-only — confirmed.** `git diff --stat` = exactly 2 files; no src/manifest/firewall/golden change; firewall passes.

### Non-blocking follow-ups (accepted)
1. Only `fitness.mean` is directly asserted (not the full robust statistic) and there is no floor on the count of `accepted` cases — transitively covered by exact per-bar returns equivalence; an explicit floor would be cleaner.
2. Roll-up covers only strictly-ascending input (out-of-order documented out-of-scope).
3. Guards mutate the reference copy, not production source (unavoidable for an external test; logically sound).
