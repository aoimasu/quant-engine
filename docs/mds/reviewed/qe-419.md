## Review verdict — [Approved]
1. **Single source of truth (core).** Server: `resolve_config_path()` (`QE_CONFIG` or `config.toml`) → `load_app_config` = `Config::load(Profile::RuntimeSim, path)` → `StorageDirs::from_config` reads `cfg.storage.{artifacts_dir,market_dir}`. Spawned CLI (`crates/cli/src/main.rs`): same `QE_CONFIG`/`config.toml` default, same RuntimeSim profile, same fields → byte-identical resolution. `ServerConfig::run_manager()` builds `CliJobSpawner::new(cli_bin).with_config_path(self.config_path)` and `spawn()` sets `cmd.env(QE_CONFIG, config_path)`, so every production child is pinned to the exact file the server loaded and guarded. No residual second source: the old `DEFAULT_ARTIFACTS_DIR`/`DEFAULT_MARKET_DIR` consts and the `ServerConfig.artifacts_dir/market_dir` fields are deleted; read-state now flows from `server_dirs`.
2. **Boot mismatch guard (core).** `check_storage_dirs_match(server, cli)` is pure (`server == cli ⇒ Ok`, else structured `Err`) and unit-tested for match/divergent-market/divergent-artifacts/no-override/divergent-override/agreeing-override. Wired in `main.rs` after config load and BEFORE `read_state()`/`bind`, returning `ExitCode::FAILURE` — never lazily at query time. No boot path skips it: the single production `read_state` consumes `server_dirs`, which only exists past the guard.
3. **Deprecation safety.** `QE_SERVER_*` still honoured via `server_storage_dirs`/`apply_server_overrides` but each set value emits a distinct deprecation `warn!`; a divergent override then trips the fatal `error!` + `FAILURE`. `agreeing_override_passes_the_guard` proves an existing deploy that set `QE_SERVER_*` equal to config still boots; `divergent_override_is_rejected_by_the_guard` proves two-source drift can't silently return.
4. **Vintage/goldens unmoved.** Diff touches no `qe-config`/`qe-vintage` schema, no `config.example.toml`, no fixture/golden — only `crates/server` code + Cargo.lock/Cargo.toml + two docs. Goldens byte-identical.
5. **Firewall.** New `qe-server → qe-config` edge legal — `firewall_holds_in_the_workspace` passes; `qe-config` (qe-domain/serde/figment/sha2) reaches no forbidden crate. One workspace-pinned dep.
6. **Panic-free / lifecycle.** No unwrap/expect on the boot/config path. QE-409 boot checks (secret policy, insecure-cookie warn) and QE-407 (reconcile-on-start, graceful drain) remain, sensibly ordered: reconcile → storage-guard → open-store → secret-check → warn → bind → drain (guarding the store before opening it). Blast radius confined to `crates/server`.

### Non-blocking (documented follow-up)
1. The guard compares the configured path STRINGS, not canonicalized absolute paths, so the invariant for RELATIVE storage dirs (e.g. `data/artifacts`) relies on the server and child sharing a CWD. They do today — the child inherits the parent's CWD (`spawn()` sets no `current_dir`) — so it is airtight in the current spawn model, and the `QE_CONFIG` pin covers config-file discovery under CWD drift. Forward-looking: add a one-line note or debug-assert so a future change that sets the child's `current_dir` can't silently reintroduce physical divergence while the string compare still passes. Deferred (no code defect today).
