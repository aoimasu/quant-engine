### Goal
A backtest needs data in the LMDB store; ingestion wasn't wired into any command (decision D3). Provide a
minimal populate path + a read-only coverage query the UI's Market-data view (QE-257) uses.

### Acceptance criteria (from backlog)
- [x] `coverage()` over the sample store returns the expected symbol/range/bar-count rows.
- [x] `ingest` populates a store from an in-memory source in a test.

### Implementation summary (plan Task 6)
- `crates/cli/src/jobs/ingest.rs`: `run_ingest(&mut impl HistoricalSource, params, progress) -> Result<(), RunError>`,
  `coverage(store, instruments) -> Result<Vec<CoverageRow{symbol,resolution,from,to,bars}>, RunError>`,
  `CoverageRow` (std/serde-only), `IngestParams`. Scans `Resolution::ALL` per instrument, min/max `open_time` + count.
- `Command::Ingest { config, start, end, resolution }` variant + `parse_args` arm + tests; `run_ingest_command`
  dispatch streaming JSON-line progress; `RunError::Ingest`. Real network ingestion gated behind the new
  default-off `http` feature (out of scope); the command emits a terminal `{"t":"error"}` naming `http` until it lands.
- `crates/cli/tests/ingest_job.rs`: `coverage_over_sample_store_reports_expected_rows` (pins from/to/bars=120 from
  independent constants), `coverage_is_empty_for_unknown_instrument`, `ingest_populates_store_from_in_memory_source`
  (populate + verify bars/funding/premium round-trip), `ingest_rejects_invalid_instrument`.

### Review ‚Äî [Approved] (0 blocking, 3 non-blocking)
- **Deviation 1 (QE-257 firewall risk) ‚Äî FOLLOW-UP, not a blocker.** No breach today (no `qe-server` firewall
  rule yet). But `coverage`/`CoverageRow` live in `qe-cli` (which pulls `qe-runtime`); a future `qe-server`

  `http` feature truly default-off and non-leaking; determinism/firewall green.
- Nits: `HistoricalWindow.base` ignored by `run_ingest` (redundant); `open_interest`/`mark_price` silently
  dropped (design-noted) ‚Äî worth an inline comment at the drop site.

### CARRIED FOLLOW-UP ‚Üí QE-257 (HARD REQUIREMENT)
**Relocate `coverage`/`CoverageRow` from `qe-cli::jobs::ingest` to `qe-storage`** (its natural layer ‚Äî body
uses only `MarketStore` + `qe-domain`, zero runtime coupling), swapping the error type `RunError` ‚Üí

`qe-server ‚äÑ qe-runtime` firewall rule that QE-254 introduces. Do it as part of QE-257.
