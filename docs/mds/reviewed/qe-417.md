## Review verdict — [Approved]
`MarkEma` gains `update_after(dt)` with per-tick `alpha = 1 − 0.5^(Δt/half_life)`. Backward-compat is byte-for-byte: `with_half_life` stores the same formula, `update` unchanged, `update_after(1.0)` recomputes the identical `≈0.011486`; proven tick-for-tick by `one_second_spacing_matches_old_fixed_alpha_series` (and `update_after_at_one_second_matches_fixed_update`), with all pre-existing MarkEma/live_mark tests unchanged → no golden churn. AC(a): 300s gap → α=0.96875 → smoothed near-reseeds (≈196.9), so the DD probe no longer runs on stale equity through a reconnect window. Panic-free on the deny-order feed (`from_f64_retain().unwrap_or(ONE).clamp(0,1)`; Δt clamped ≥0 → α=0 on dup/out-of-order/skew; half_life≤0 → α=1; first sample seeds). AC(b): `MarkTick.stale = prev.is_some() && Δt > staleness_bound_secs` (default 5.0s, first tick false). Deterministic (integer `event_time_ms` Δt, no wall-clock, `f64::powf`); new `stale` field's blast radius contained (no external struct-literal/exhaustive match).

Non-blocking follow-ups (both fold into **QE-429**):
1. `stale` is surfaced on every `MarkTick` but no consumer acts on it yet — wire a breaker halt / cockpit annotate on a stalled feed (latent until the live runtime-risk loop is wired).
2. `staleness_bound_secs`/`tick_secs`/`half_life` are constructor params with exposed defaults, not yet threaded to a Config/TOML knob — promote to a runtime-risk config block (same theme as the QE-416 constants) so operators can tune without a recompile.
