## Review (pass 1) — Approved

Careful principal-level pass on `BootstrapPipeline::replay` composing QE-201/205/206/207 into a
deterministic cold start.

**(A) Determinism (the AC) — holds.** `replay` is a pure function of `(window, vintage, cfg, tiers)`: no
clock, no RNG. Dedup is `sort_by_key(open_time)` (stable) + a strict-increasing filter; the merge timeline
is a `Vec` ordered by a stable `sort_by_key((ts, ord))`; the evaluator iterates `Vec`s. **No `HashMap` is
ever iterated in the replay path.** `cold_start_is_deterministic` runs `replay` twice and asserts identical
decisions/`bars_replayed`/`last_mark_price`/`vintage_id`, non-vacuous via `has_enter_and_exit`.

**(B) "State == continuously-running planner" — property holds.** `reconstructed_state_equals_a_continuous_
run` bootstraps `[0..n)`, `go_live()`s, feeds bar `n`, and asserts it decides identically to a continuous
session fed `[0..n]` then bar `n` — proving the bootstrap's dedup + replay + `go_live` boundary leave state
exactly where a continuous run holds it. Non-blocking note: the continuous reference re-implements the same
timeline construction inline, so it pins bootstrap-vs-continuous *equivalence* but not the as-of *ordering
correctness* independently — addressed in pass 2.

**(C) As-of merge ordering — correct by construction.** Context ordinal 0 < bar ordinal 1 + stable sort
places context at `ts == bar.open_time` before the bar (QE-206's inclusive `value.ts <= bar.open_time`).
The fixture exercises between-bars (funding at `2·MIN`) and on-boundary (funding at `90·MIN == bar 18`)
cases; `dedup_stitches_overlapping_pages` and `pagination_is_invariant…` prove order/duplication
independence.

**(D) Paginated REST path — genuinely exercises retry.** `paginate_klines`/`paginate_series` wrap
`VenueRestClient::paginate` with a caller-injected `decode`. `pagination_is_invariant_to_page_splits_and_
retries` serves the same 6 bars as 1 vs 3 pages with a scripted `RestError::Transient` between pages — the
client backs off (advancing `TestClock`) and recovers, so the transient is truly on-path; both decode and
reconstruct identically. The decode-closure injection keeps the Binance JSON schema out of `qe-runtime` —
the right boundary (concrete decoder deferred to QE-219, mirroring the `http`-gated transport).

**(E) Scoping (markPrice, thin history) — defensible.** markPrice is replayed at 1-min cadence but is not a
catalogue feature (flow context is funding/OI/premium); surfaced as `last_mark_price` for QE-210/211. The
`Event::Mark` arm never touches the factor join. `thin_history_holds_without_spurious_entries` (3 bars) →
all-`Hold`, no panic — a legitimate cold-start edge.

**(F) Architecture — no new crate edge.** `Cargo.toml` adds only `thiserror`; `dependency_topology` +
firewall green; deny clean.

## Review (pass 2) — Approved

Pass-2 commit `775d295` addresses the pass-1 note (B). `replay_matches_an_independent_as_of_oracle` builds
the expected stream by a structurally different construction: per-bar **pull** as-of
(`obs.filter(|t| *t <= bar_ts).max_by_key(ts)`, observed before `on_bar`) versus `replay`'s global
event-stream **push** merge (ordinals + one stable sort). They share no ordinal/sort code, so a wrong
ordinal/mis-sort would land context on the wrong bar and diverge — the bug can no longer hide. The `<=`
matches QE-206 and `max_by_key`'s last-of-equal-ts matches the push merge's insertion-order last-wins.
Pure test addition (+46 lines inside `mod tests`), no production code touched. No regression: qe-runtime
24→25, workspace 482→483, decoupling + firewall green, clippy/fmt/deny clean. A clean close to the
single-command-orchestration chain.
