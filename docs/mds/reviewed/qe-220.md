### Goal
*(Reviewer-added.)* If reconstructed peak/state diverges from continuous state, every drawdown breaker is
mis-anchored — a capital-risk event. Assert bootstrap-reconstructed state equals continuously-running state on
the breaker-relevant fields (committed peak, dormancy latches, positions).

### Acceptance criteria (from backlog)
- [x] Reconstructed vs continuous state match bit-for-bit on breaker-relevant fields —
  `reconstructed_state_matches_continuous_bit_for_bit`.

### Implementation summary
- Integration test `crates/runtime/tests/restart_parity.rs` (no production surface). **Reconstructed** =
  `ReconstructedState::from_replay(...)` (production restart path). **Continuous** = an **independent**
  reference (plain running max for the peak, plain entered-flag for dormancy, session's final positions —
  never calls `CommittedPeak`/`from_replay`), so parity is a real check, not a tautology.
- Scenario: a `cycling_genome` (trades → active, non-flat) + an all-off genome (dormant) over a real
  `EvaluatorSession` (40 bars, past warmup); **peak-at-index-0** equity paths with a length-20 monotonic
  decline (longer than any plausible breaker window), so a windowed regression of any realistic size is caught.

### Review notes — [Approved] (two passes)

**Pass 1 (@ `57a416ca`) — [Reviewed].** Verified sound: the continuous reference is genuinely independent (hand-
rolled max + entered-flag, never calls the production path); non-vacuousness properly guarded
(`dormancy_latches_match_for_traded_and_untraded` asserts strategy 0 `!is_dormant()` **and** `position !=
flat()`, so a warmup change that stopped it trading fails loudly). **F1 (blocker):** the length-3 interior-peak
path (`[100, 150, 120]`) only caught a `window == 1` regression — for any `W ≥ 2` the window still contains the
index-1 peak, so a realistically-sized windowed `from_replay` regression would compute the same 150 and pass.
Fix: put the true max at index 0 with a decline longer than any plausible breaker window.

**Pass 2 (@ `6666b988`) — [Approved].** F1 resolved: equity paths are now `declining_from(peak, 2)` — a
monotonically-declining length-20 path with the true max at index 0 — and
`committed_peak_is_true_all_time_max_not_trailing` loops `window in 1..EQUITY_LEN` asserting every
trailing-window max is `< 150`. Since the last `w` elements (any `w < 20`) exclude index 0, a windowed
regression of any realistic size diverges from the plain-max reference and fails parity — closing exactly the
gap raised. Parity + dormancy assertions logically unchanged and still non-vacuous. All gates green. Approved.
