## Decision summary

`StrategyRepository::try_record` admits a candidate IFF `QualityGate::persists` (QE-114) ‚Äî Exploration
candidates (`n < min_exploitation_windows`) are rejected regardless of mean, below-LCB exploitation
candidates rejected, survivors appended with a `Lineage` (QE-006). `record_survivors` is the batch form;
`write_jsonl`/`read_records` round-trip records as JSON Lines. `NoiseRobustFitness` gained serde derives
(additive; persisted records always finite).

## Review (pass 1) ‚Äî Approved

Principal-level review of `c3ccd97`. Both halves of the AC met ‚Äî early-lucky/below-bar candidates can't be
persisted and every persisted record carries a resolvable lineage that survives the round-trip. No real bugs.

- **(a) Early-lucky guarantee ‚Äî delegated, no bypass.** `try_record` records iff `gate.persists`;
  `records.push` happens only inside `try_record` (which `record_survivors` routes through), so no
  admission path bypasses the gate. The QE-114 gate rejects any Exploration (`n < min`) candidate before
  checking the mean, plus below-LCB exploitation candidates. The test drives the *real* QE-120 backtester
  for both the n=6 survivor and the n=1 lucky case (same genome).
- **(b) Lineage.** Mandatory `Lineage` field (record unconstructible without it), `HasLineage` resolves
  it, `Lineage::id()` stable; JSONL round-trip reproduces records exactly with matching lineage id.
- **(c) serde.** Additive derives; persisted records finite on every real path (gate enforces
  `mean.is_finite()`; non-finite `std_error` screened by the LCB; backtester yields `sqrt ‚â• 0`);
  round-trip lossless, blank lines skipped.
- **(d) `record_survivors`** folds `try_record` in input order, persists exactly the survivors (holds 2
  and 7 in order, n=1 and below-bar dropped).
- **(e)** serde‚Üí`io::Error::other`, I/O via `?`, no unwrap/index in non-test code, empty repo handled.

Gates re-run locally green (qe-wfo 84, clippy/fmt clean, `cargo deny` ok ‚Äî `serde_json` correctly promoted
